## 1. train.py 全景

`train.py` 是整个项目的"导演"。我们先看它宏观上做了什么，再钻进循环细节。

### 1.1 main 入口：解析参数 → 读 config → safe_state → init GUI → training

`train.py` 的 `__main__` 段（`train.py:201` 起）做了五件事：

**密度化（densification）** 是 3DGS 的核心机制之一，比喻说就是"哪里细节不够，就在哪里再撒一把点"（`train.py:126-136`）：

```python
if iteration < opt.densify_until_iter:                      # 密度化只在前期进行
    gaussians.max_radii2D[visibility_filter] = torch.max(...)  # 记录每个点出现过的最大半径
    gaussians.add_densification_stats(...)                    # 累加视图空间梯度
    if iteration > opt.densify_from_iter and iteration % opt.densification_interval == 0:
        size_threshold = 20 if iteration > opt.opacity_reset_interval else None
        gaussians.densify_and_prune(opt.densify_grad_threshold, 0.005,
                                    scene.cameras_extent, size_threshold)
    if iteration % opt.opacity_reset_interval == 0 or (dataset.white_background and iteration == opt.densify_from_iter):
        gaussians.reset_opacity()                             # 周期性把不透明度打回低值，防止"老点"霸占
```

直观理解三个阈值：

- `densify_grad_threshold=0.0002`：哪个高斯在视图空间"位置梯度"大（说明它落在错误地方、被强烈推拉），就**复制/分裂**它。
- `0.005` 是 opacity prune 阈值：太透明（几乎看不见）的点直接**剪掉**。
- `size_threshold=20`：屏幕上太大的高斯（over-sized）也剪掉，避免一个大高斯糊住一片。

> 一个容易忽略的点：`densify_and_prune` 之后高斯数量会变，`_objects_dc`（身份编码）也跟着一起被复制/裁剪（见 `scene/gaussian_model.py:518-580`），所以分裂出的"子高斯"会继承父高斯的身份暗号。

## 2. 关键新增：分类器 classifier

Gaussian-Grouping 相比原版 3DGS 最大的改动，就是给每个高斯额外加了一组 **身份编码（Identity Encoding）**。它和球谐颜色 SH 是平行的概念——SH 决定高斯"长什么颜色"，身份编码决定高斯"属于哪个物体"。

具体地，模型里有 `gaussians._objects_dc`（`scene/gaussian_model.py:51`），存盘时的形状是 `(N, 1, num_objects=16)`，即每个 3D 点有 16 维的"暗号"。注意它和 SH 一样经过了 `RGB2SH` 变换并做了转置（`scene/gaussian_model.py:139-158`），所以在内部存储里通道维是 1、特征维是 16。`num_objects=16` 由 `gaussians.num_objects` 固定为 16（`scene/gaussian_model.py:52`）。

但要把它变成可解释的"我是第几号物体"，还需要一个**分类器（classifier）**（`train.py:33`）：

```python
classifier = torch.nn.Conv2d(gaussians.num_objects, num_classes, kernel_size=1)
```

### 为什么是 `Conv2d(..., kernel_size=1)`？

一个 1×1 卷积核的本质就是**逐像素的线性层**：它不看邻域，只对每个空间位置的通道维做一次矩阵乘法。

- 输入 `objects`（即渲染出来的身份特征图 `render_pkg["render_object"]`）：形状 `(1, 16, H, W)`。
- 输出 `logits`：每个像素在 `num_classes` 个类别上的得分，形状 `(1, num_classes, H, W)`。

这跟对每个像素跑一个 `Linear(16, num_classes)` 完全等价，但用卷积实现既快又能直接吃二维特征图。

> 比喻：16 维身份编码是每个像素自带的"暗号"，分类器就是"翻译官"，把暗号翻成"我是第几号物体"的概率分布。其中 `num_objects=16` 由 `gaussians.num_objects` 决定，`num_classes` 是物体类别总数（`train.json` 里设为 256，会通过 `dataset.num_classes` 传进来）。

配套的损失与优化器（`train.py:34-35`）：

```python
cls_criterion = torch.nn.CrossEntropyLoss(reduction='none')
cls_optimizer = torch.optim.Adam(classifier.parameters(), lr=5e-4)
```

注意分类器有**自己独立的优化器**（学习率 5e-4），和高斯的优化器分开 step、分开 `zero_grad`（`train.py:140-143`）。这是因为分类器是个普通的小神经网络，而高斯属性有各自专门的学习率调度。两者通过同一个 `loss.backward()` 联动求导，但各自走自己的优化器更新参数。

## 3. 三个损失（重点）

### 3.1 (a) 重建损失：L1 + λ·(1 − SSIM)

```python
gt_image = viewpoint_cam.original_image.cuda()
Ll1 = l1_loss(image, gt_image)
loss = (1.0 - opt.lambda_dssim) * Ll1
     + opt.lambda_dssim * (1.0 - ssim(image, gt_image)) + ...
```

其中 `lambda_dssim = 0.2`（`arguments/__init__.py:85`，定义在 `OptimizationParams` 里）。

写成公式：

```math
L_{\text{recon}} = (1-\lambda)\,L_1(\hat I, I) + \lambda\,(1-\text{SSIM}(\hat I, I)),\qquad \lambda=0.2
```

- $L_1$ 是逐像素绝对差均值，**对小误差敏感、数值稳定**，但容易让图像偏模糊。
- SSIM（结构相似性）衡量亮度、对比度、结构三方面的相似，更贴近人眼感受，但对细节过度敏感、训练初期不稳。

二者混合（且 SSIM 取 `1-` 让"越大越差"）正是为了**取长补短**：L1 保底，SSIM 提锐度。λ=0.2 是 3DGS 原文的经验值。

### 3.2 (b) 分组损失 loss_obj（`train.py:87-90`）

```python
gt_obj   = viewpoint_cam.objects.cuda().long()      # 2D 物体 id 标签图
logits   = classifier(objects)                      # (1, num_classes, H, W)
loss_obj = cls_criterion(logits.unsqueeze(0), gt_obj.unsqueeze(0)).squeeze().mean()
loss_obj = loss_obj / torch.log(torch.tensor(num_classes))   # 归一化到 (0,1)
```

逐行拆解：

1. `viewpoint_cam.objects` 是这张训练图的 **2D mask 标签**，每个像素值就是该像素所属物体的 id（第 8 节细讲它怎么来）。
2. `classifier(objects)` 把 16 维身份特征图翻成 `num_classes` 维 logit。
3. `CrossEntropyLoss(reduction='none')` 逐像素算交叉熵，再 `.squeeze().mean()` 得到标量。
4. 最后除以 `log(num_classes)`：

```math
L_{\text{obj}} = \frac{\text{CrossEntropy}(\text{logits},\ \text{gt\_obj})}{\log(C)}
```

**为什么要除以 $\log C$？** 当真实类别是均匀猜测时，交叉熵的理论上界约为 $\log C$（$C$ 是类别数）。除以它就把 loss 压缩到约 $(0,1)$ 区间，和重建损失（也是 0~1 量级）量纲匹配，避免分组损失因为类别数多而数值过大、压过重建项。

### 3.3 (c) 3D 一致性正则 loss_cls_3d（每 reg3d_interval 步一次，`train.py:97-102`）

```python
if iteration % opt.reg3d_interval == 0:
    logits3d   = classifier(gaussians._objects_dc.permute(2,0,1))      # (1, C, N)
    prob_obj3d = torch.softmax(logits3d, dim=0).squeeze().permute(1,0) # (N, C)
    loss_obj_3d = loss_cls_3d(
        gaussians._xyz.squeeze().detach(),   # 3D 坐标，detach！
        prob_obj3d,
        opt.reg3d_k, opt.reg3d_lambda_val,
        opt.reg3d_max_points, opt.reg3d_sample_size)
    loss = (1.0 - opt.lambda_dssim) * Ll1 + ... + loss_obj + loss_obj_3d
```

> 张量形状小提示：`_objects_dc` 存盘是 `(N, 1, 16)`，`.permute(2,0,1)` 后变成 `(16, 1, N)`，正好当作一张 `C_in=16`、宽为 N 的"图像"喂给分类器，得到 `(num_classes, 1, N)`，squeeze 掉第 1 维并转置成 `(N, num_classes)` 的概率矩阵。这样分类器既能吃渲染的 2D 身份特征图，也能直接吃 3D 身份编码，**复用同一组权重**。

这一步**直接对 3D 点本��**做约束（而不是对渲染图），核心思想：**空间上邻近的 3D 点，其身份分布也应该相近**。这正是下一节要逐行精讲的 `loss_cls_3d`。

> 关键细节：`gaussians._xyz.squeeze().detach()` 里的 `detach()` 说明这个 loss **只回传到身份编码和分类器**，不会挪动点的 3D 位置——否则正则项会把点全拽到一起，破坏几何。

最终总损失：

```math
L = L_{\text{recon}} + L_{\text{obj}} + L_{\text{3D-consist}}
```

（非 reg3d 步只有前两项。）

## 4. loss_cls_3d 逐行精讲（`utils/loss_utils.py:74`）

先把签名和完整实现贴出来：

```python
def loss_cls_3d(features, predictions, k=5, lambda_val=2.0,
                max_points=200000, sample_size=800):
    # 1. 下采样
    if features.size(0) > max_points:
        indices = torch.randperm(features.size(0))[:max_points]
        features    = features[indices]
        predictions = predictions[indices]

    # 2. 随机采样 sample_size 个"查询点"
    indices         = torch.randperm(features.size(0))[:sample_size]
    sample_features = features[indices]
    sample_preds    = predictions[indices]

    # 3. 算两两距离 + top-k 近邻
    dists                  = torch.cdist(sample_features, features)   # (sample_size, N')
    _, neighbor_indices    = dists.topk(k, largest=False)             # (sample_size, k)
    neighbor_preds         = predictions[neighbor_indices]            # (sample_size, k, C)

    # 4. KL 散度
    kl = sample_preds.unsqueeze(1) * (
            torch.log(sample_preds.unsqueeze(1) + 1e-10)
          - torch.log(neighbor_preds + 1e-10))
    loss = kl.sum(dim=-1).mean()

    # 5. 归一化 + 乘权重
    num_classes      = predictions.size(1)
    normalized_loss  = loss / num_classes
    return lambda_val * normalized_loss
```

注意签名里的默认值 `sample_size=800`、`max_points=200000` 只是兜底；实际训练时由 `train.json` 通过 `opt.reg3d_sample_size=1000`、`opt.reg3d_max_points=200000` 传入覆盖（见第 5 节）。

### 4.1 步骤 1：下采样 max_points

如果点云规模 $N > \text{max\_points}$，就**随机抽 max_points 个点**保留。原因纯粹是显存与算力：下一步 `torch.cdist` 是 $O(N^2)$ 的，点太多会爆显存。

### 4.2 步骤 2：随机采样 sample_size 个查询点

再从（可能已下采样的）点云里**随机抽 sample_size 个点**作为"查询点"——只用它们去算 loss，是蒙特卡洛式的近似，进一步降低开销。记这一批查询点为：

$$
S = \{p_1, p_2, \dots, p_{\text{sample\_size}}\}
$$

每个点有坐标 $x_i \in \mathbb{R}^3$（这里 `features` 就是 3D 坐标 `_xyz`）和身份分布 $q_i \in \Delta^{C-1}$（`predictions`，softmax 后的概率向量，$C$ 是类别数）。

### 4.3 步骤 3：cdist + top-k 近邻

`torch.cdist(sample_features, features)` 计算 sample 中每个点 $p_i$ 到**全体**（下采样后）点 $p_j$ 的欧氏距离矩阵，形状 `(sample_size, N')`。

然后 `topk(k, largest=False)` 取**距离最小的 k 个**作为 $p_i$ 的邻居集合 $\mathcal{N}_k(p_i)$，拿到它们对应的预测分布 $\{q_j\}_{j\in\mathcal{N}_k(p_i)}$。

### 4.4 步骤 4：KL 散度

核心公式：

```math
\text{KL}(q_i \,\|\, q_j) = \sum_{c=1}^{C} q_i(c)\,\bigl(\log q_i(c) - \log q_j(c)\bigr)
```

代码里 `kl` 张量形状是 `(sample_size, k, C)`，`kl.sum(dim=-1)` 在类别维求和得到每个 (查询点, 邻居) 对的 KL 值，形状 `(sample_size, k)`；`.mean()` 把 sample_size×k 全部平均，得到最终标量。`1e-10` 是防止 `log(0)` 的数值稳定项。

**为什么用 KL 而不是别的（比如 MSE）？**

KL 散度天然适合"两个概率分布"的差异度量，且对**高置信度的预测**更敏感：如果 $q_i$ 很确定（某个 $q_i(c)\to 1$），而邻居 $q_j(c)$ 很小，KL 会给一个很大的惩罚，强有力地把相邻点的身份"拉齐"。这正符合我们想要的物理直觉：**同一物理表面的邻近点应当属于同一物体**。MSE 对概率向量不够自然，也不带这种"信息论"含义。

### 4.5 步骤 5：归一化 + 权重

```math
L_{\text{3D}} = \lambda \cdot \frac{1}{C}\,\mathbb{E}_{i,\,j\in\mathcal{N}_k(p_i)}\bigl[\text{KL}(q_i \| q_j)\bigr]
```

除以类别数 $C$ 是粗略的归一化（因为 KL 的上界随 $C$ 增长），让 loss 大致落在 $(0,1)$ 区间；再乘以 `lambda_val`（`train.json` 里 = 2）作为该项的总权重。

> 关于 `detach`：调用方传入的 `features` 是 `gaussians._xyz.squeeze().detach()`，意味着坐标在算距离时**不参与求导**；但 `predictions` 来自 `prob_obj3d`（含分类器和身份编码），是带梯度的。所以梯度只会流向 `_objects_dc` 和 `classifier`，把"邻居身份趋同"的压力施加在身份编码上，而不会去挪动 3D 坐标——这正是我们想要的。


逐项解读与调参直觉：

| 参数 | 值 | 含义 | 调参直觉 |
|---|---|---|---|
| `densify_until_iter` | 10000 | 在此步之前才允许密度化；之后高斯数量冻结 | 想让点云更精细可调大；显存吃紧可调小 |
| `num_classes` | 256 | 物体类别总数（含背景） | 必须大于场景里实际物体数；设太大浪费容量，太小会"撞类别" |
| `reg3d_interval` | 5 | 每 5 步算一次 3D 一致性正则 | 越小越频繁、约束越强、训练越慢；过大约束稀疏 |
| `reg3d_k` | 5 | 每个 query 点取最近 5 个邻居 | 大 k 更平滑但慢；小 k 更局部 |
| `reg3d_lambda_val` | 2 | 3D 一致性项的权重 | 物体边界"过糊"可调大；细节被磨平调小 |
| `reg3d_max_points` | 200000 | 下采样上限，超过就随机抽 | 显存不够调小；点云小时无影响 |
| `reg3d_sample_size` | 1000 | 每次算 loss 抽多少 query 点 | 越大越准但越慢；1000 是经验上的平衡点 |

> 注意：`arguments/__init__.py` 里 `OptimizationParams` 的默认值（`densify_until_iter=15000`、`reg3d_interval=2`、`reg3d_max_points=300000`）和这份 JSON 不一样；`ModelParams` 里 `num_classes` 默认是 200。**真正生效的是 JSON 的值**，因为它在 main 里覆盖了 `args`。

## 6. Scene 类（`scene/__init__.py`）

`Scene` 是数据与高斯模型的"组装车间"。构造函数 `Scene.__init__`（`scene/__init__.py:25`）按如下流程走：

```text
1. 判断 load_iteration：
     None      → 训练模式（从 SfM 点云开始）
     -1 / 整数 → 加载模式（读取已训练的 point_cloud/iteration_<n>/point_cloud.ply）
2. 判断数据集类型：
     存在 sparse/                → Colmap（真实场景）
     存在 transforms_train.json  → Blender（合成场景）
3. 调对应 reader（readColmapSceneInfo / readNerfSyntheticInfo）拿到 scene_info
4. 训练模式下：拷贝 input.ply、写出 cameras.json
5. cameras_extent = scene_info.nerf_normalization["radius"]   # 场景"半径"
6. 构造 train/test 相机列表（cameraList_from_camInfos）
7. 高斯初始化：
     加载模式 → gaussians.load_ply(...)
     训练模式 → gaussians.create_from_pcd(scene_info.point_cloud, cameras_extent)
```

关键判断分支（`scene/__init__.py:43-49`）：

```python
if os.path.exists(os.path.join(args.source_path, "sparse")):
    scene_info = sceneLoadTypeCallbacks["Colmap"](args.source_path, args.images,
                args.eval, args.object_path, n_views=args.n_views,
                random_init=args.random_init, train_split=args.train_split)
elif os.path.exists(os.path.join(args.source_path, "transforms_train.json")):
    scene_info = sceneLoadTypeCallbacks["Blender"](...)
else:
    assert False, "Could not recognize scene type!"
```

`cameras_extent`（`scene/__init__.py:69`）就是 `nerf_normalization["radius"]`，它有两个用途：

- 作为高斯的 `spatial_lr_scale`（位置学习率的尺度因子）；
- 作为密度化/剪枝时的"屏幕大小"参考阈值。

如果是训练模式（非加载），还会做两件事（`scene/__init__.py:51-63`）：

- 把 SfM 点云 `scene_info.ply_path` 复制到 `model_path/input.ply`；
- 把所有相机序列化成 `model_path/cameras.json`（用 `camera_to_JSON`）。

对外接口：

```python
scene.save(iteration)             # 把当前高斯存成 point_cloud/iteration_<n>/point_cloud.ply
scene.getTrainCameras(scale=1.0)  # 取训练相机列表
scene.getTestCameras(scale=1.0)   # 取测试相机列表
```

## 7. readColmapSceneInfo（重点，`scene/dataset_readers.py:132`）

真实场景靠 COLMAP 做 SfM 重建，产物在 `sparse/0/` 下：

- `cameras.bin`：相机内参（焦距、分辨率、相机模型）。
- `images.bin`：每张图的外参（四元数 qvec + 平移 tvec）。
- `points3D.bin`：稀疏 3D 点云（带颜色）。

### 7.1 读内外参 + 关联 mask

```python
cam_extrinsics = read_extrinsics_binary(.../images.bin)   # 每张图的 R, T
cam_intrinsics = read_intrinsics_binary(.../cameras.bin)  # 焦距、宽高
cam_infos = readColmapCameras(cam_extrinsics, cam_intrinsics,
                              images_folder, objects_folder)
```

`readColmapCameras`（`scene/dataset_readers.py:66`）对每张图：

1. 用内参算出 `FovX / FovY`（通过 `focal2fov`），只支持 `SIMPLE_PINHOLE` / `PINHOLE` 两种相机模型。
2. 把 qvec 转成旋转矩阵，注意 `R = np.transpose(qvec2rotmat(extr.qvec))`——这是为了匹配 CUDA 端 `glm` 的存储约定（R 存转置）。
3. **关键关联**（`dataset_readers.py:98-99`）：

```python
object_path = os.path.join(objects_folder, image_name + '.png')
objects     = Image.open(object_path) if os.path.exists(object_path) else None
```

也就是：每张训练图 `images/xxx.jpg` 都对应一张 `object_mask/xxx.png`，后者就是这张图的 2D 物体 id 标签图（每个像素值 = 该像素所属物体的 id）。这是分组监督的来源。如果该图找不到 mask，`objects` 就是 `None`。

### 7.2 train/test 划分

`readColmapSceneInfo` 的划分行为取决于 `eval` 和 `train_split` 两个开关（`dataset_readers.py:149-194`）：

- **`eval=True`**（要留测试集）：
  - `train_split=True`：读取 `images_train/` 目录里的文件名清单，**名单内进 train，名单外进 test**。这是 Gaussian-Grouping 推荐的"显式划分"方式。
  - `train_split=False`：经典 LLFF 风格的 `llffhold=8`，即**每 8 张取 1 张做测试**，其余进 train；之后再按 `n_views` 在 train 集上做子采样：

    ```python
    train_cam_infos = [c for idx, c in enumerate(cam_infos) if idx % llffhold != 0]
    test_cam_infos  = [c for idx, c in enumerate(cam_infos) if idx % llffhold == 0]
    ```

  - `n_views` 子采样（仅在 `train_split=False` 时生效）：

    | `n_views` | 行为 |
    |---|---|
    | 100（默认） | 不抽样 |
    | 50 | 用 `np.linspace` 抽 50% 视角 |
    | 任意整数 N | 均匀抽 N 个视角 |

- **`eval=False`**（不留测试集）：所有图进 train，test 为空；若 `train_split=True` 则只读 `images_train/` 名单内的图。

### 7.3 points3D → .ply 与 getNerfppNorm

如果 `random_init=False`，会优先用 `sparse/0/points3D.ply`；若该 `.ply` 不存在，就用 `read_points3D_binary` 读 `.bin` 并 `storePly` 转成 `.ply`（`dataset_readers.py:211-221`），方便后续 `fetchPly` 一次性载入。若 `random_init=True`，则生成一个 10 万个随机点的点云（`dataset_readers.py:198-209`），用于无 SfM 的合成场景。

`getNerfppNorm`（`dataset_readers.py:43`）计算所有训练相机光心的平均位置 `center` 和最远距离 `diagonal`，得到：

```python
radius   = diagonal * 1.1     # 留 10% 余量
translate = -center           # 把场景平移到原点
```

返回 `{"translate": ..., "radius": ...}`。其中 `radius` 就是上面提到的 `cameras_extent`，决定了空间学习率缩放和密度化阈值；`translate` 在本仓库的 `getWorld2View2` 中也会用到（用于把场景居中）。

## 8. Camera 类（`scene/cameras.py`）

`Camera`（`scene/cameras.py:18`）封装一张图对应的所有渲染所需信息。最核心的是几个**预计算好的变换矩阵**（`scene/cameras.py:55-58`）：

```python
self.world_view_transform = torch.tensor(getWorld2View2(R, T, trans, scale)).transpose(0,1).cuda()
self.projection_matrix   = getProjectionMatrix(znear, zfar, FoVx, FoVy).transpose(0,1).cuda()
self.full_proj_transform  = (self.world_view_transform.unsqueeze(0)
                              .bmm(self.projection_matrix.unsqueeze(0))).squeeze(0)
self.camera_center        = self.world_view_transform.inverse()[3, :3]
```

逐个解释（注意 `.transpose(0,1)` 是为了匹配 CUDA 光栅化器期望的**列主序**存储）：

| 字段 | 含义 | 怎么来的 |
|---|---|---|
| `original_image` | GT RGB 图 `(3, H, W)`，已 clamp 到 [0,1] | `image.clamp(0,1)` |
| `FoVx / FoVy` | 水平/垂直视场角 | 从焦距 + 宽高用 `focal2fov` 算 |
| `znear / zfar` | 近/远裁剪面 | 固定 `0.01 / 100.0` |
| `world_view_transform` | 世界→相机变换 W2C（4×4） | `getWorld2View2(R, T, trans, scale)` |
| `projection_matrix` | 相机→裁剪空间投影 | `getProjectionMatrix(znear=0.01, zfar=100, FoVx, FoVy)` |
| `full_proj_transform` | 世界→裁剪空间的合并矩阵 | `W2C @ Projection` |
| `camera_center` | 相机在世界坐标系的位置 | `W2C^{-1}` 的最后一行前三列 |

> 一句话理解：渲染时，3D 高斯的位置先用 `full_proj_transform` 投到屏幕，再做 EWA splatting；`camera_center` 用于计算高斯朝向相机的视角相关颜色。

**Gaussian-Grouping 新增的 `objects` 字段**（`scene/cameras.py:60-63`）：

```python
if objects is not None:
    self.objects = objects.to(self.data_device)   # 物体 id 标签图
else:
    self.objects = None
```

它就是第 7.1 节读进来的 2D mask 标签图，正是 `loss_obj` 的监督信号 `gt_obj`（在 `train.py:87` 处 `.long()` 转成整数）。

## 9. loadCam / PILtoTorch（`utils/camera_utils.py`）

`loadCam`（`utils/camera_utils.py:20`）负责把 `CameraInfo`（PIL 图 + 各种参数）转换成可训练的 `Camera` 对象，关键是**分辨率策略**（`camera_utils.py:23-40`）：

```python
if args.resolution in [1, 2, 4, 8]:
    resolution = round(orig_w/(resolution_scale*args.resolution)), round(...)
else:                                  # -1 或自定义宽度的"目标长边"
    if args.resolution == -1:
        if orig_w > 1600:              # 长边超过 1600 自动降采样
            global_down = orig_w / 1600
        else:
            global_down = 1
    else:
        global_down = orig_w / args.resolution
    scale = float(global_down) * float(resolution_scale)
    resolution = (int(orig_w/scale), int(orig_h/scale))
```

直觉：

- `-r 1/2/4/8`：直接整数倍下采样。
- `-r -1`（默认）：**长边 > 1600 自动降到 1600**，避免大图爆显存（会打一条 warning，见 `camera_utils.py:29-32`）。
- `-r <自定义宽度>`：把长边缩到指定宽度。

随后 `PILtoTorch` 把 PIL 图转成 `(C, H, W)` 的 torch tensor，RGB 通道 + 可能的 alpha 通道（`camera_utils.py:42-48`）：

```python
resized_image_rgb = PILtoTorch(cam_info.image, resolution)
gt_image   = resized_image_rgb[:3, ...]
loaded_mask = resized_image_rgb[3:4, ...] if resized_image_rgb.shape[1]==4 else None
```

最后构造 `Camera`，**关键改动**是把 mask 标签也一起塞进去（`camera_utils.py:54`）：

```python
return Camera(..., objects=torch.from_numpy(np.array(cam_info.objects)))
```

`torch.from_numpy(np.array(...))` 把 PIL mask 直接转成整型 tensor，不归一化（因为物体 id 是离散标签，不是连续颜色）。

## 11. 训练输出目录结构

训练完成后，`output/<scene>/`（或用 `-m` 指定的 `model_path`）下大致是这样：

```text
output/<scene>/
├── cfg_args                                    # 训练时所有参数的 Namespace 字符串
├── input.ply                                   # 原始 SfM 点云（来自 sparse/0/points3D.ply）
├── cameras.json                                # 所有相机的内外参序列化
├── point_cloud/
│   └── iteration_30000/
│       ├── point_cloud.ply                     # 训练好的高斯（含身份编码 _objects_dc）
│       └── classifier.pth                      # 分类器权重（Gaussian-Grouping 新增）
└── chkpnt<n>.pth                               # 可选：断点续训用的 checkpoint
```

几个要点：

- `point_cloud.ply` 由 `scene.save(iteration)` → `gaussians.save_ply(...)` 写出（`scene/__init__.py:91-93`），里面除了 3DGS 原有的 SH/opacity/scaling/rotation，还**额外存了每个高斯的 16 维身份编码**。
- `classifier.pth` 由 `torch.save(classifier.state_dict(), ...)` 单独存（`train.py:123`），推理时必须和高斯一起加载，否则没法把身份编码翻成类别概率。
- `cfg_args` 是 `prepare_output_and_logger` 写的（`train.py:160`），内容是 `str(Namespace(**vars(args)))`，渲染时由 `get_combined_args` 用 `eval` 读回来。
- `input.ply` 和 `cameras.json` 只在**训练模式**（非 load）下生成（`scene/__init__.py:51-63`）。

## 小结

本节我们把 Gaussian-Grouping 的训练侧代码"从入口到磁盘文件"完整走了一遍：

- `train.py` 的主循环在原版 3DGS 基础上**多渲染了一份身份特征图 `render_object`**，并用一个 1×1 卷积 `classifier` 把它翻成类别 logit。
- 损失由三部分组成：**重建损失**（L1 + 0.2·(1−SSIM)）保证图像质量；**分组损失 loss_obj** 用交叉熵把渲染出的身份对齐到 2D mask 标签；**3D 一致性正则 loss_cls_3d** 用 KL 散度强制空间邻近点身份趋同（且通过 `detach` 不挪动坐标）。
- 配置文件 `config/gaussian_dataset/train.json` 提供了 `densify_until_iter=10000`、`num_classes=256` 等关键超参，会在 main 里覆盖默认值。
- 数据侧由 `Scene` → `readColmapSceneInfo` → `Camera` → `loadCam` 流水线完成：读 COLMAP 内外参、关联 `object_mask/` 标签、计算场景半径、按分辨率策略下采样，最终生成训练相机列表。
- 最终产物 `point_cloud.ply` + `classifier.pth` 是推理（渲染任意视角的物体分割/编辑）所必需的两个文件。

读完本节，你应该能独立读懂 `train.py` 的每一行，并在需要时调整 `train.json` 里的超参来适配自己的数据集。下一节我们将进入推理与渲染代码，看这些产物如何被用来生成新视角的 RGB 图与物体 mask。
