## 1. 三大编辑能力总览

在前几节里，我们花了很大力气给每一个 3D 高斯贴上"身份编码（Identity Encoding）"，并训练一个分类器（classifier）把它判成"这是苹果 / 那是杯子"。一旦这件事做完，整张高斯点云就被切成了一块块"可单独操作的对象"，编辑能力也就水到渠成了。

本仓库实现了三种编辑，复杂度递增：

| 能力 | 入口脚本 | 一句话描述 | 比喻 |
|------|----------|------------|------|
| 物体移除（removal） | `edit_object_removal.py` | 把某�� id 的所有高斯直接删掉 | 从水母群中抽走一只 |
| 物体修复（inpaint） | `edit_object_inpaint.py` | 删掉物体后，在被挖空的地方补上背景 | 抠掉图章后，用周围纸的纹理把洞填平 |
| 开放词汇分割 | `render_lerf_mask.py` + `ext/grounded_sam.py` | 输入"red apple"这样的文本，输出这个物体的二值 mask | 用一句话指挥分类器："给我所有红苹果" |

它们共享同一条核心思路：**先用分类器在 3D 点上算出"属于哪些 id"的概率，再用一个阈值得到 3D mask，最后对 mask 内的点是删除、是补种、还是染色，取决于具体任务。** 接下来我们逐个拆解。

## 2. 3D 物体移除 `edit_object_removal.py`

### 2.1 主流程 `removal()`

入口函数 `removal()`（`edit_object_removal.py:151`）只做三件事，干净利落：

1. **加载**：重建 `GaussianModel` 与 `Scene`，加载高斯点云，并用一个 `1×1` 卷积重建并加载分类器权重 `classifier.pth`（`:153-161`）。注意分类器是现搭的：`torch.nn.Conv2d(gaussians.num_objects, num_classes, kernel_size=1)`（`:157`），它把 16 维身份特征映射成 `num_classes` 个类别的 logits。
2. **移除**：调用 `removal_setup()` 把目标物体的高斯从点云里剔除（`:164`）。
3. **渲染验证**：重新加载移除后的点云，用 `render_set()` 渲染 train/test 视角看效果（`:167-173`）。

要删哪个物体由 json 配置文件决定，三个默认值在脚本末尾（`:202-204`）：`num_classes=200`、`removal_thresh=0.3`、`select_obj_id=[34]`。

### 2.2 怎么得到 3D 点 mask：`removal_setup()`

最关键的一步是把"哪个物体"翻译成"哪些点"。看 `removal_setup()`（`edit_object_removal.py:71-92`）：

```python
logits3d = classifier(gaussians._objects_dc.permute(2,0,1))   # :74
prob_obj3d = torch.softmax(logits3d, dim=0)                   # :75
mask = prob_obj3d[selected_obj_ids, :, :] > removal_thresh    # :76
mask3d = mask.any(dim=0).squeeze()                            # :77
```

逐步解释：

- `gaussians._objects_dc` 的形状是 `(N, 1, 16)`，是每个高斯的 16 维身份特征（`N` 是点的总数）。
- `.permute(2,0,1)` 把它整成 `(16, 1, N)`，假装是一张"通道=类别、空间=N 个点"的图，喂给那个 `1×1` 卷积分 类器。
- `classifier` 输出 logits `(num_classes, 1, N)`，再 `softmax` 沿类别维归一化，得到每个点属于每个 id 的概率。
- `selected_obj_ids` 是你想删的物体 id 列表（来自 json 配置，如 `[34]`）。取出这些通道，判断 `> removal_thresh`（默认 0.3），再用 `.any(dim=0)` 把"属于任意一个被选 id"的点合并成一张 `(N,)` 的布尔 mask。

一句话：**"分类器说哪些点大概率是要删的物体，就把它们标 True。"**

### 2.3 为什么要再加一层凸包：`points_inside_convex_hull()`

光按身份判还不够稳。想想一只玩具熊：它表面看得到的高斯身份学得很准，但**藏在它肚子内部、被自己挡住的那些点**，因为训练时几乎从没被任何视角"看见并监督"过，身份编码可能学得很糊，分类器判不出它是熊。

补救办法（`edit_object_removal.py:31-69`）：既然这些点在物理空间上确实被熊"包住"，那就**用已经判为熊的表面点求一个凸包（convex hull），把落在凸包内部的点一并纳入**。

具体分三步：

1. **IQR 去离群点**（`:51-56`）：先对已选中的点按每个坐标轴算四分位距 IQR，把偏离过远的点（比如个别飞到天上的噪点）当离群点剔除，避免凸包被拉得奇形怪状。

   ```math
   \mathrm{IQR}=Q_3-Q_1,\quad \text{离群}:\ x<Q_1-1.0\cdot\mathrm{IQR}\ \text{或}\ x>Q_3+1.0\cdot\mathrm{IQR}
   ```

   其中 $Q_1,Q_3$ 是该坐标轴上所有选中点的第 25 / 第 75 百分位，系数 `outlier_factor=1.0`（`:79` 调用时传入）。

2. **Delaunay 三角化求凸包**（`:61`）：用 `scipy.spatial.Delaunay` 给过滤后的点建三角网，三角网的最外圈就定义了凸包。
3. **判断每个点是否在凸包内**（`:64`）：`delaunay.find_simplex(pts) >= 0` 一行搞定——返回 `-1` 表示点在所有单纯形之外（即凸包外），`>=0` 表示在内部。

最后（`:79-80`）把"身份 mask"和"凸包 mask"取并集：

```python
mask3d_convex = points_inside_convex_hull(gaussians._xyz.detach(), mask3d, outlier_factor=1.0)
mask3d = torch.logical_or(mask3d, mask3d_convex)
```

注意这里传入 `points_inside_convex_hull` 的是 `mask3d`（表面点），但它会在**全部点云**上判断是否落进这个凸包，所以能把内部漏掉的点捞回来。

### 2.4 真正的删除发生在 `gaussian_model.py`

拿到最终的 `mask3d` 后，调用 `gaussians.removal_setup(opt, mask3d)`（`edit_object_removal.py:85`），实现在 `scene/gaussian_model.py:216-241`。它的做法是先把 mask 取反（`:218`，`mask3d = ~mask3d.bool().squeeze()`），即选中"要保留的背景点"，再用这个反向 mask 把七项属性（`_xyz`、`_features_dc/rest`、`_opacity`、`_scaling`、`_rotation`、`_objects_dc`）从原张量里切片出来（`:221-227`），最后重建为新的 `nn.Parameter`，且**剩下的点全部 `requires_grad=False` 冻结**（`:235-241`）。因为移除是个一次性的"硬编辑"，不需要再训练。结果随后存到 `point_cloud_object_removal/iteration_{xxx}/point_cloud.ply`（`:89-90`）。

## 3. 3D 物体修复 `edit_object_inpaint.py`（最复杂的一节）

### 3.1 三步总流程

物体修复不是单独一个脚本就能完成的，它在论文里是三步流水线：

1. **移除物体**（用上一节的 `edit_object_removal.py`）——把熊删掉，留个洞。
2. **2D 修复**：用一个预训练的 2D 修复模型 **LaMa**，在每张训练图上把"熊挡住的那块区域"用周围像素补上，得到一张看起来没熊的照片，并产出一个 `object_mask` 标明"哪片像素是被补出来的"。
3. **3D 微调**（**本脚本负责**）：把这些 2D 修复后的图当作"伪标签（pseudo label）"，在被挖空的地方种出新的高斯，让它们学着把背景长出来。

> 注意：第 2 步的 LaMa 不在这个仓库里，它的 2D 修复结果会被存成新的"训练图 + object_mask"。`edit_object_inpaint.py` 只管第 3 步。脚本末尾会把数据路径切到修复结果上：`dataset.object_path = 'object_mask'`、`dataset.images = 'images'`（`edit_object_inpaint.py:195-196`），随后重新加载 `Scene` 来渲染验证。

### 3.2 定位待补区域：`finetune_inpaint()` 开头

`finetune_inpaint()`（`edit_object_inpaint.py:53`）的前半段和移除几乎一模一样：算 `mask3d`（`:57-60`），再叠凸包（`:62-63`）。区别在下一步调的是 `inpaint_setup` 而不是 `removal_setup`。这之后还会初始化一个冻结的 LPIPS 网络（`:70-73`），用作感知损失。

### 3.3 在空洞里种新点：`inpaint_setup()` + `initialize_new_features()`

`gaussians.inpaint_setup(opt, mask3d)`（调用在 `edit_object_inpaint.py:67`，实现在 `scene/gaussian_model.py:244`）做两件事：

**(a) 冻结旧点，保留它们当"老师"。** 同样先把 mask 取反（`:289`），抽出背景点（不在 mask 内、即未被删除的部分），切片保留并 `requires_grad=False`（`:293-299`）。

**(b) 在被挖空的区域"种"新点，并用周围邻居初始化它们。** 这是关键，看内嵌函数 `initialize_new_features()`（`scene/gaussian_model.py:246-287`）：

- 被挖掉的那些位置 `mask_xyz_values`（`:290`，即 `self._xyz[~mask3d]`，是要删除点的原始坐标）就是新点的候选坐标。
- 用 **KDTree**（`cKDTree`）在剩余背景点里找每个新点的 **k 近邻（默认 k=5）**（`:260`、`:267`）。
- 新点的每一项属性（xyz、颜色 DC/rest、不透明度、缩放、旋转、身份编码）都取这 5 个邻居的**均值**（`:271-285`，对 2D/3D 张量分别用 `feature_np[selected_indices]` / `[..., :, :]` 索引，再 `np.mean(axis=1)`）。

这是个很合理的初始化：新长出来的高斯一开始"长得像周围的背景"，后续微调才容易收敛。最后把这些新点 `torch.cat` 到旧点后面（`:322-328`），且**只有新点 `requires_grad=True`**，相当于"只让新学生练习，老学生保持不动"。同时重置 `max_radii2D`、`xyz_gradient_accum`、`denom` 和优化器（`:331-339`）。

### 3.4 两项损失：masked L1 + LPIPS

微调循环（`edit_object_inpaint.py:76-113`）每步随机抽一张视角，渲染后算两个损失。

**第一个：masked L1（`:84`）。**

```python
mask2d = viewpoint_cam.objects > 128            # :82 当前视角里"被 LaMa 修复过"的区域
gt_image = viewpoint_cam.original_image.cuda()  # :83
Ll1 = masked_l1_loss(image, gt_image, ~mask2d)  # :84 只在"非修复区"算 L1
```

注意这里传进去的 mask 是 `~mask2d`（取反）。回顾 `masked_l1_loss`（`utils/loss_utils.py:18-22`）：它把 mask 广播到与图像同通道数，只在 `mask=1` 的位置累加 $|image-gt|$，再除以 mask 的总和归一化。也就是说：**让新点去拟合那些"原始就能看见的背景像素"**（`objects<=128` 的区域），而不是去拟合 LaMa 凭空补出来的内容（那部分可能不准，留给下面的 LPIPS 管）。

**第二个：LPIPS 感知损失（`:86-92`）。** LPIPS 用一个预训练 VGG 网络衡量两张图在"深层特征"上的差异，比 L1 更懂"看起来像不像"。为了让它聚焦在修复区域，做了三步裁剪：

```python
bbox = mask_to_bbox(mask2d)                                       # :86 求修复区的最小外接矩形
cropped_image = crop_using_bbox(image, bbox)                      # :87
K = 2
rendering_patches = divide_into_patches(cropped_image[None,...], K)   # :90 切 2x2
lpips_loss = LPIPS(rendering_patches.squeeze()*2-1, gt_patches...).mean()  # :92
```

- `mask_to_bbox`（`:32-39`）：把散乱的 mask 收紧成一个矩形框（min/max 行列）。
- `divide_into_patches(image, K=2)`（`:46-51`）：用 `F.unfold` 把裁剪后的图切成 $2\times2=4$ 块。
- **为什么切 patch？** 因为 LPIPS 在大图上算的是整体相似度，而修复只发生在局部。切成小块后，每块都更"局部"，VGG 提取的特征更贴近人眼对纹理的判断，损失更敏锐。最后对 4 块取均值。`*2-1` 是把 `[0,1]` 的图像搬到 LPIPS 期望的 `[-1,1]` 区间。

总损失（`:94`）：

```math
\mathcal{L}=(1-\lambda_{\text{dssim}})\,\mathcal{L}_1+\lambda_{\text{dssim}}\,\mathcal{L}_{\text{LPIPS}}
```

其中 `lambda_dssim` 默认 0.5（配置里写成 `lambda_dlpips`，从 `edit_object_inpaint.py:239` 读入，赋给 `opt.lambda_dssim`）。

### 3.5 微调时还在长点：densify_and_prune

注意 `:97-105`：在前 5000 步里，每 300 步还会做一次 `densify_and_prune`（密度化与剪枝）。这就像"哪里细节不够就在哪里再撒一把点"——新种的高斯如果某些地方太稀，就再分裂；太胖的就剪掉。5000 步后停止密度化，专心把颜色和形状磨平。微调总步数默认 10000（`finetune_iteration`，`:240`）。结果存到 `point_cloud_object_inpaint/iteration_{xxx}/`（`:116-117`）。

## 4. 开放词汇分割 `render_lerf_mask.py` + `ext/grounded_sam.py`

### 4.1 痛点：SAM 不认识"苹果"

前面所有编辑都要你先告诉它 `select_obj_id`（比如 `[34]`）。但用户哪知道 34 是苹果还是杯子？用户只会说"red apple"。问题就变成：**怎么把一句文本翻译成一个物体 id？**

直接用 SAM（Segment Anything Model）也不行：SAM 是"类别无关（class-agnostic）"的——它能把前景物体抠出来，但分不清是苹果还是鼻子。所以本仓库用了 **Grounded-SAM**，把"开词汇检测"和"精确分割"串起来。

### 4.2 Grounded-SAM：文本 → 框 → mask

核心函数 `grouned_sam_output()`（`ext/grounded_sam.py:58-99`）两步走：

1. **Grounding DINO 出框**（`:69-75`）：给它一张图和一句文本（如 `"red apple"`），它输出所有匹配物体的边界框 `boxes`，以及每个框的置信度 `logits` 和文本短语 `phrases`。两个默认阈值 `BOX_TRESHOLD=0.3`、`TEXT_TRESHOLD=0.45`（`:58`）用来过滤低质量框。
2. **SAM 把框变成像素级 mask**（`:80-92`）：`sam_predictor.set_image` 编码图像，把框转成像素坐标（`:83`），再喂给 `predict_torch`，得到精确的物体 mask。

一个文本可能匹配到多个框（桌上好几个苹果），最后把它们的 mask 求并（`:99`）：

```python
return torch.sum(masks, dim=0).squeeze().bool(), annotated_frame_with_mask
```

### 4.3 文本 mask → 物体 id：`select_obj_ioa()`

拿到文本对应的 2D mask 后，还要知道它对应分类器里的哪个 id。`select_obj_ioa()`（`ext/grounded_sam.py:102-115`）输入的是首帧的 **id 整数图**（`argmax` 得到的 `pred_obj`，见 `render_lerf_mask.py:45`），用一个叫 **IoA（Intersection-over-Area）** 的指标：

```math
\mathrm{IoA}=\frac{|C\cap M|}{|C|}
```

其中 $C$ 是某个 id 在 id 图上占的像素集合，$M$ 是文本 mask。对每个候选 id 算 IoA，**>0.7 就选中**（`ioa_thresh=0.7`，`:110-112`）。直观说：如果一个 id 的像素有大半都被文本 mask 盖住了，就认定"这个 id 就是用户说的物体"。

> 为什么用 IoA 而不是 IoU？因为文本 mask 可能只覆盖物体的一部分（比如只框了苹果的正面），但 id 区域是整个苹果。用 IoU（分母是并集）会偏低而误杀；用"id 区域里有多少被盖住"的 IoA（分母是 $|C|$）更宽容。

这一步只在**首帧**做一次（`render_lerf_mask.py:41-50`）：用首帧渲染图跑 Grounded-SAM 得到文本 mask，再和首帧分类器预测的 id 图算 IoA，确定 `selected_obj_ids`。

### 4.4 把首帧定下的 id 推广到所有帧

有了 `selected_obj_ids`，后续每帧（`:52-79`）不再需要 Grounded-SAM，只做（阈值 `threshold=0.2`，函数签名 `:30`）：

```python
prob = torch.softmax(logits, dim=0)
pred_obj_mask = prob[selected_obj_ids, :, :] > threshold   # :64 threshold=0.2
pred_obj_mask = pred_obj_mask.any(dim=0)                   # :65 任意被选 id 命中即为正
```

输出每帧该物体的二值 mask（`*255` 存成 uint8，`:66`），存到 `test_mask/{帧号}/{文本prompt}.png`（`:53-54`、`:75`）。

### 4.5 换场景怎么改

`render_lerf_mask.py:114-121` 写死了三个 LERF 场景的文本 prompt 列表（其他场景直接 `raise NotImplementedError`）：

```python
if 'figurines' in dataset.model_path:
    positive_input = "green apple;green toy chair;old camera;porcelain hand;red apple;red toy chair;rubber duck with red hat"
elif 'ramen' in dataset.model_path:
    positive_input = "chopsticks;egg;glass of water;pork belly;wavy noodles in bowl;yellow bowl"
elif 'teatime' in dataset.model_path:
    positive_input = "apple;bag of cookies;coffee mug;cookies on a plate;paper napkin;plate;sheep;spoon handle;stuffed bear;tea in a glass"
```

每个 prompt 用 `;` 分隔，`positives = positive_input.split(";")`（`:123`），循环对每个词渲染一次（`:126-130`）。**换成你自己的场景，就在这里加一个 `elif`（替换那个 `raise`）写上你想分割的物体名称。**

## 5. 可视化工具 `render.py`

训练完总要"眼见为实"。`render.py` 提供两个上色函数和一套渲染流水线。

### 5.1 `feature_to_rgb()`：16 维特征 → 3 色彩图

身份编码是 16 维的，没法直接看。`feature_to_rgb()`（`render.py:26-45`）用 **PCA（主成分分析）** 把 16 维压到 3 维，再归一化到 `0-255` 当 RGB：

```python
features_reshaped = features.view(features.shape[0], -1).T   # (H*W, 16)  :31
pca = PCA(n_components=3)
pca_result = pca.fit_transform(features_reshaped.cpu().numpy())  # (H*W, 3)  :35
```

效果是"每个物体一种颜色"，相近的身份在 PCA 空间里也靠近，看起来花花绿绿，方便人眼分辨不同物体。

### 5.2 `id2rgb()` + `visualize_obj()`：整数 id → 高对比色

分类器输出的是整数 id（0,1,2,...），`id2rgb()`（`render.py:47-65`）用一个巧妙技巧上色——**黄金比例**：

```python
golden_ratio = 1.6180339887
h = ((id * golden_ratio) % 1)   # 把 id 映射到色相 hue  :53
```

因为黄金比例是无理数，相邻 id 的色相会"均匀散开"在色环上，于是 id=1 和 id=2 的颜色对比很强，不会撞色。`id=0` 是无效区域，返回黑色（`:60-61`）。饱和度和亮度还按 `id%2` 在 0.5/1.0 间交替（`:54`）。`visualize_obj()`（`:67-73`）就是遍历每个出现的 id，把整张标签图涂成彩色。

### 5.3 `render_set()`：五列拼成视频

`render_set()`（`render.py:76-129`）对每个视角渲染并保存五样东西，最后横向拼成一张大图、再串成 `result.mp4`：

| 列 | 含义 | 来源 |
|----|------|------|
| `gt` | 真值照片 | `view.original_image`（`:105`） |
| `rgb` | 高斯渲染的 RGB | `results["render"]`（`:90`） |
| `gt_obj` | 真值物体彩图 | `visualize_obj(view.objects)`（`:99`） |
| `pred_obj` | 预测物体彩图 | `visualize_obj(argmax(classifier))`（`:95`） |
| `render_obj` | 16 维特征 PCA 图 | `feature_to_rgb(rendering_obj)`（`:101`） |

拼接发生在 `:123`：`np.hstack([gt,rgb,gt_obj,pred_obj,render_obj])`，视频宽度是单图宽度的 5 倍（`:112`）。（顺带一提：`render.py` 用 `ours_{}` 带下划线作输出目录名，而两个编辑脚本里同名函数用的是 `ours{}` 不带下划线，属于历史遗留，不影响功能。）

## 6. 评估

### 6.1 新视角合成指标：`metrics.py`

`metrics.py` 算的是 3DGS 系列的标准三件套（`:71-78`）：

- **PSNR（峰值信噪比）**：像素级误差，越高越好（`utils/image_utils.py` 的 `psnr`）。
- **SSIM（结构相似性）**：衡量亮度/对比度/结构的一致性，越接近 1 越好（`utils/loss_utils.py` 的 `ssim`）。
- **LPIPS**：用 VGG 深层特征算感知差异，越低越好（`lpipsPyTorch` 的 `lpips(..., net_type='vgg')`，`:74`）。

它遍历 `test/{method}/renders` 和 `gt` 下的同名图片，逐对算指标后取均值（`:71-74`），存进 `results.json` 和 `per_view.json`（`:88-91`）。这套指标衡量的是"渲染出来的照片像不像真的"，用于报告移除/修复后的整体画质。

### 6.2 分割指标：`script/eval_lerf_mask.py`

开放词汇分割输出的是二值 mask，要用分割专用指标。这个脚本算两个：

**IoU（Intersection-over-Union，交并比）**（`calculate_iou`，`:66-73`）：

```math
\mathrm{IoU}=\frac{|G\cap D|}{|G\cup D|}
```

其中 $G$ 是真值 mask，$D$ 是预测 mask。是分割任务最经典的指标。

**Boundary-IoU（边界交并比）**（`boundary_iou`，`:36-53`）：只比较两个 mask 的**边界**重不重。核心是 `mask_to_boundary()`（`:15-33`）：先用一个 $3\times3$ 核对 mask 做**腐蚀（erode）**，再用"原 mask 减腐蚀后 mask"得到一圈边界像素：

```python
new_mask_erode = cv2.erode(new_mask, kernel, iterations=dilation)  # :30
return mask - mask_erode                                          # :33
```

腐蚀强度 `dilation` 由 `dilation_ratio=0.02` 乘以图像对角线长度决定（`:23-24`）。然后在边界上算 IoU（`:50-52`）。

> **为什么要单独看 Boundary-IoU？** 普通 IoU 对"大块区域重叠"很敏感，但对"边界是不是锐利"几乎无感——一个 mask 哪怕整体偏移几像素，IoU 也可能还很高。而 Gaussian-Grouping 的卖点之一就是"物体边界比点云级别的同类方法更清晰"。Boundary-IoU 专门盯边界，正好能把这个优势量化出来，所以是论文重点报告的指标。

脚本最后按类别（每个文本 prompt 当一类，`cat_id` 来自文件名）统计均值（`:111-112`），再算总体均值（`:115-116`，对各类均值再取平均）。

## 7. 小结：编辑能力都来自"分组"

回头看，本节所有花哨的操作——删掉一只熊、把熊的位置补成桌面、用一句话"red apple"圈出苹果——本质上都依赖同一件事：**训练阶段给每个高斯学到了靠谱的身份编码，且分类器能把它判成具体的物体 id。** 一旦有了这个"分组"，编辑就退化成三步：

1. 用分类器 + 阈值拿到 3D 点 mask；
2. （可选）用凸包把内部漏掉的点补全；
3. 对 mask 内的点是删、是补、还是染色，按任务选择。

评估侧也是同样道理：渲染出图就用 PSNR/SSIM/LPIPS 评画质，渲染出 mask 就用 IoU/Boundary-IoU 评分割。理解了这条主线，你就掌握了 Gaussian-Grouping 从"训练"走向"可用"的全部接口。
