## 1. 3DGS 三分钟速成（新手必读）

后面的文档默认你知道"一个 3D 高斯是什么"。这一节把这件事讲透。

### 1.1 一个 3D 高斯 = 一颗"半透明、可变形、会发光的小水母"

Gaussian Grouping 是在 **3D Gaussian Splatting（3DGS）** 上改造的。3DGS 的核心想法非常直白：

> **不再用一个大神经网络去"记住"整个场景（这是 NeRF 的做法），而是用几十万、上百万个"小水母"摆满 3D 空间，每个小水母负责一小块颜色和亮度。渲染时，把它们一次性"泼（splat）"到屏幕上。**

每颗"小水母"就是一个 **3D 高斯（Gaussian）**，它由这几个属性决定（对应代码 `scene/gaussian_model.py` 里的字段）：

| 属性 | 代码字段 | 直观理解 |
| --- | --- | --- |
| 中心位置 | `_xyz`（N,3） | 这颗水母飘在 3D 空间的**哪里** |
| 大小/形状 | `_scaling`（N,3）+ `_rotation`（N,4） | 水母**有多胖、被拉成什么椭圆、朝哪个方向**（缩放 + 四元数旋转） |
| 不透明度 | `_opacity`（N,1） | 水母**有多透明**（sigmoid 前的值） |
| 颜色 | `_features_dc` / `_features_rest` | 水母**发什么光**，而且这个光会**随你看它的角度变化**（用球谐函数 SH 表示） |

> 把"中心 + 大小 + 朝向"组合起来，数学上等价于一个**椭球**的协方差矩阵 Σ。后面数学篇会严格推导。

### 1.2 为什么颜色要用"球谐函数（SH）"？

因为同一个点从不同角度看，颜色可能不同（比如金属的反光、镜面）。**球谐函数（Spherical Harmonics, SH）** 是一组定义在球面上的"标准基底函数"，可以把"随方向变化的颜色"展开成一组系数。

- 3DGS 用 SH 阶数 ≤ 3，每个颜色通道有 `(3+1)² = 16` 个系数。
- 第 0 个系数叫 **DC（直流）项**，存的是"平均颜色"；其余 15 个叫 **rest（高阶）项**，存"随视角的细微变化"。
- 代码里因此拆成两个字段：`_features_dc`（常数项）和 `_features_rest`（高阶项）。

> 一句话记忆：**`_features_dc` = 这颗水母的本色，`_features_rest` = 换个角度看到的色差。**

`utils/sh_utils.py` 里的 `RGB2SH` / `SH2RGB` 负责在"RGB 颜色"和"SH 系数"之间换算，常数 `C0 = 0.28209479177387814` 就是零阶球谐基函数的值。

### 1.3 渲染：把这些水母"泼"成一张 2D 图

给定一个相机视角，3DGS 用 **EWA splatting** 把每颗 3D 水母投影到屏幕上，变成一颗 2D 的半透明水母；然后按**从前到后的顺序**做 **alpha 合成（alpha blending）**：

```math
C(u)=\sum_i c_i\,\alpha_i\prod_{j<i}(1-\alpha_j)
```

- $c_i$：第 $i$ 颗水母的颜色；
- $\alpha_i$：它在像素 $u$ 处的"有效不透明度"（= 自身不透明度 × 它在这像素处的高斯密度）；
- $\prod_{j<i}(1-\alpha_j)$：前面所有水母"挡住"它的程度。

**关键**：整个过程是**可微（differentiable）**的——梯度能从"渲染出的图"一路传回到每颗水母的 `_xyz/_scaling/_rotation/_opacity/颜色`。所以才能用"渲染图和真实图的差"当损失，反向传播去优化这些水母。

### 1.4 为什么要"密度化（densification）"？

初始那批水母（来自 COLMAP 的稀疏点云）远远不够精细：有的地方太稀、细节糊；有的水母太大、占一大片。训练中 3DGS 会**动态增删水母**，这就是**密度化（densification）**：

- **克隆（clone）**：某个小水母周围梯度很大（说明它"不够用"），复制一份挪开。
- **分裂（split）**：某个大水母梯度很大（说明它"太大装不下细节"），拆成两颗缩小。
- **剪枝（prune）**：太透明（基本看不见）或太大的水母，删掉。

> 比喻：密度化就是"哪里细节不够就在哪里再撒一把点，哪里点没用就扫掉"。

### 1.5 Gaussian Grouping 在 3DGS 上加了什么？

3DGS 只会"画图"，不懂"这颗水母属于椅子还是桌子"。Gaussian Grouping 给每颗水母**再加一个属性**：

| 新属性 | 代码字段 | 含义 |
| --- | --- | --- |
| 身份编码 | `_objects_dc`（N,1,16） | 一个 **16 维向量**，相当于"我是哪个物体"的暗号 |

然后用两招把它训对（详见 `01`、`03`、`06`）：

1. **用 SAM 的 2D mask 当伪标签监督**：每张训练图，让 SAM/DEVA 分割出物体 mask；训练时把 3D 身份编码也"泼"成一张 16 通道图，经一个小分类器（1×1 卷积）转成"每像素属于哪个物体"的概率，与 mask 算交叉熵。
2. **3D 空间一致性正则**：3D 里相邻的点应该属于同一物体，用 k 近邻 + KL 散度约束。

训完，每颗水母既会画图、又知道自己属于谁——于是可以**按物体单独取出来编辑**（移除/修复/风格化）。

## 2. 统一术语表（全书对齐）

后面 6 篇里这些词会反复出现，含义以本表为准：

| 统一术语（中文） | 英文 / 代码 | 一句话解释 |
| --- | --- | --- |
| 高斯 / 水母 | Gaussian | 一颗 3D 半透明点 |
| 身份编码 | Identity Encoding / `_objects_dc` | 每颗高斯额外的 16 维向量，表示"属于哪个物体"。**不是** SH 颜色 |
| 身份特征图 | object feature map / `render_object` | 渲染出的 16 通道图（身份编码被 alpha 合成到 2D） |
| 分类器 | classifier | 一个 1×1 卷积（16→num_classes），把身份特征图翻译成"每像素是第几号物体" |
| 伪标签 | pseudo label | SAM/DEVA 自动产出的 2D 物体 mask，当训练的监督信号 |
| 密度化 | densification | clone / split / prune 高斯的过程 |
| 球谐函数 | Spherical Harmonics, SH | 表示"随视角变化的颜色"的一组基底 |
| EWA splatting | 弹性体泼溅 | 把 3D 高斯投影成 2D 高斯并 alpha 合成的方法 |
| 凸包 | convex hull | 物体移除时，用选中点的外围凸多边形把"内部被挡的点"也一并纳入 |
| num_classes | num_classes | 分类器输出通道数，必须 **大于** mask 里最大的物体 id |
| num_objects | `num_objects=16` | 身份编码的维度（固定 16），**不要和 num_classes 混淆** |

## 3. 五个关键事实（新手最容易误解的地方）

这五条是审稿时反复被点出的"坑"，每条都会**直接决定你能不能读懂代码 / 跑通复现**。它们散落在各篇里，这里集中讲一次。

### 事实 ①：身份编码是"伪 SH"——借了 SH 的壳，渲染时却**不走** SH

这是整个方法最容易被误读的一点，必须讲清：

- **存储上**：身份编码存成 `_objects_dc`，形状 `(N, 1, 16)`，**长得和颜色的 DC 项一模一样**（`scene/gaussian_model.py:51,158`）。
- **初始化上**：用 `RGB2SH(torch.rand(...))` 初始化（`scene/gaussian_model.py:139`），**借用了 SH 的换算通道**。
- **但渲染时**：颜色要走 `SH → RGB`（`+0.5`、乘 `C0`）才变成真实颜色；而**身份编码不走这一步**！它在修改版光栅化器里被**原样 alpha 合成**，不做 `SH2RGB`、不加 `+0.5`、不 clamp（见 `submodules/diff-gaussian-rasterization/.../__init__.py` 与底层 CUDA `forward.cu`）。

> 一句话：**身份编码只是"借用 SH 的存储格式和初始化约定"，本质是一个 16 维特征向量，渲染时和颜色走同一条 alpha 合成管线，但没有任何 SH 数学变换。** 别看到 `_objects_dc` 就以为它要 `SH2RGB`。

### 事实 ②：身份特征图**没有背景色**，而 RGB 有

- RGB 渲染：最后会加上背景色（`out_color += T * bg_color`，白或黑）。
- 身份特征图：**纯 alpha 累加，不加任何背景**（`out_objects` 不参与背景填充）。

所以渲染出的 16 通道特征图里，"没被任何高斯覆盖的区域"是 0，不是背景色。做 PCA 可视化（`render.py:26` 的 `feature_to_rgb`）时，别奇怪为什么空白区是一种特定颜色——那是 PCA 归一化后的 0。

### 事实 ③：训练有**两个独立优化器**

`train.py` 里其实有两个优化器，**分开 step / zero_grad**：

- `gaussians.optimizer`：优化所有高斯参数（xyz、颜色、scaling、rotation、opacity、**身份编码**）。
- `cls_optimizer`：单独优化分类器（1×1 卷积），学习率固定 `5e-4`，不走高斯那套指数衰减。

```python
gaussians.optimizer.step(); gaussians.optimizer.zero_grad(set_to_none=True)  # train.py:140-141
cls_optimizer.step();      cls_optimizer.zero_grad()                         # train.py:142-143
```

而且**密度化时只重建高斯优化器的状态**（`_prune_optimizer` / `cat_tensors_to_optimizer` 只动高斯那 7 组参数），分类器参数不受影响。新手照抄"单优化器"模板会漏掉 `cls_optimizer.step()`，导致分类器永远不更新。

### 事实 ④：`lambda_dssim` 这个名字在训练和修复里**含义不同**（命名陷阱）

- 在 `train.py:102`，`lambda_dssim`（默认 `0.2`）是 **D-SSIM 权重**，重建损失里 `(1-λ)·L1 + λ·(1-SSIM)`。
- 在 `edit_object_inpaint.py:94`，**同一个变量名** `lambda_dssim` 被用来装 **LPIPS 权重**（值来自 config 的 `lambda_dlpips=0.5`，见 `edit_object_inpaint.py:239`）。

> 即：`lambda_dssim` 在训练脚本里 = D-SSIM 权重；在修复脚本里 = LPIPS 权重。**变量名相同、语义完全不同**，读代码时务必结合所在文件判断。

### 事实 ⑤：`reg3d_*` 参数有"两套默认值"，以 config 为准

- `utils/loss_utils.py:74` 的函数签名默认：`max_points=200000, sample_size=800`。
- 但 `train.py:239-240` 从 `config.json` 读默认：`reg3d_max_points=300000, reg3d_sample_size=1000`。
- 实际 `config/gaussian_dataset/train.json` 里写的是 `200000 / 1000`。

**结论**：真正训练时跑的是 **config.json 的值**（`train.py` 把它传进函数，覆盖函数签名默认）。函数签名的 `200000/800` 只在"完全不传参"时才生效。改参数时改 config，别改函数签名。

另外：3D 一致性正则**不是每步都算**，而是**每 `reg3d_interval`（config 里是 5，代码默认 2）步触发一次**（`train.py:97`）。只有触发的那些步，总损失里才包含 `loss_obj_3d`。

## 4. 复现前还要知道的两件事

1. **本仓库不用 git submodule**。`submodules/` 下两个目录（`diff-gaussian-rasterization`、`simple-knn`）是**直接提交进主仓库**的，`git clone` 时会一起下载，**不需要** `--recursive`，也**不要**去 `pip install` 官方版 `diff-gaussian-rasterization`（否则渲染会报 `unexpected keyword argument 'sh_objs'`）。详见 `05_复现指南.md` 第 2 节。

2. **`num_classes` 要大于 mask 里的最大物体 id**。DEVA 产出的物体 id 可能很大，所以项目统一用 `256`。自制数据时若 id 超过 256，要么调大 `num_classes`，要么在 DEVA 里压缩 id。注意区分：`num_classes`（分类器输出类别数，可配）≠ `num_objects=16`（身份编码维度，固定）。
