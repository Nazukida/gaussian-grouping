## 1. 这个项目是什么

一句话定义：

> **Gaussian Grouping（高斯分组）= 3D 高斯泼溅（重建）+ 物体级分割 + 场景编辑。**

它来自 ETH Zurich 等机构，发表于 ECCV 2024。论文标题是《Gaussian Grouping: Segment and Edit Anything in 3D Scenes》。相关链接：

- 论文：https://arxiv.org/abs/2312.00732
- 项目主页：https://ymq2017.github.io/gaussian-grouping

要理解它，先得理解它站在谁的肩膀上。

### 背景：新视角合成（Novel View Synthesis）

想象你拿手机绕着一座雕像拍了一圈照片。**新视角合成**要做的就是：根据这些照片，在电脑里"还原"出这座雕像的 3D 模型，然后**任意换个角度，都能渲染（render，即把 3D 场景画成 2D 图片）出一张逼真的新照片**。这就像是给场景建一个"虚拟摄影棚"。

这条路上有两个里程碑：

| 方法 | 表示方式 | 速度与质量 |
|---|---|---|
| **NeRF**（Neural Radiance Field，神经辐射场） | 把场景存在一个神经网络（隐式、连续的函数）里 | 质量好，但渲染很慢（每张图要逐像素查询网络） |
| **3D Gaussian Splatting（3DGS）** | 把场景表示成**几十万个显式的、半透明的小点（高斯椭球）** | 既快（实时渲染）又高质量 |

打个比方：NeRF 像一张"藏起来的地图"，要看哪一角得现场画给你；3DGS 则像把场景拆成一堆"半透明、可变形、会发光的小水母"直接摆在 3D 空间里，渲染时一次性把它们"泼（splat）"到屏幕上，所以又快又好。

Gaussian Grouping 正是在 3DGS 的基础上，给它"开窍"——让它不仅会画图，还懂"哪个点属于哪个物体"。

---

## 2. 它想解决什么问题

原版 3DGS 虽然画图又快又美，但它**只懂得"长得什么样"**——也就是外观（颜色）和几何（位置、形状），完全不懂**物体级的语义**。

举个具体的痛点：3DGS 重建出来的上百万个高斯点里，有一团属于"椅子"、一团属于"桌子"、一团属于"地板"。可是模型根本不知道这件事，在它眼里所有点都一样。正因为"分不清谁是谁"，你**没法单独把椅子删掉、没法把桌子的颜色改成红色、也没法修复被遮挡的区域**——因为一动手就不知道该动哪些点。

Gaussian Grouping 的目标，就是给 3DGS 补上这份"物体级理解（object-level understanding）"，从而打开 3D 场景编辑的大门。

---

## 3. 核心思想（重点）

它的做法可以拆成三个互相配合的招式。我们先讲思路，数学细节留到专门的推导篇。

### 招式一：给每个高斯贴一张"身份标签"——身份编码（Identity Encoding）

原版 3DGS 里，每个高斯点（你可以把它想象成一只小水母）携带这几样属性：

- **位置**：它飘在 3D 空间的哪里（中心坐标）；
- **形状/大小**：它有多胖、被拉成什么椭圆（协方差，由缩放和旋转决定）；
- **颜色**：它发什么光（用球谐函数 Spherical Harmonics, SH 表示，能呈现不同视角下颜色变化）；
- **不透明度**：它有多透明。

Gaussian Grouping 给每个高斯**额外加了一个属性**，叫做**身份编码（Identity Encoding）**。在代码里它是一个长度为 **16** 的向量（`self.num_objects = 16` 见 `scene/gaussian_model.py:52`；初始化见 `scene/gaussian_model.py:138-140`）。

> 比喻：身份编码就像给每只小水母贴了一张"我是哪个物体"的小名牌。名牌本身只是 16 个数字，但训练之后，同一把椅子上的水母名牌会越来越像，桌子上的名牌则自成一类。最终，靠"名牌相似度"就能把点分成一组一组的物体。

这里有两个细节值得留意，对零基础读者很有帮助：

1. **它不直接写成"椅子=3、桌子=7"这种人类可读的整数**，而是一个需要学习、可微（能跟模型一起被梯度优化）的连续向量。这样它才能融入可微渲染的训练流程。
2. **初始化时它甚至借用了 SH 的工具箱**：代码用 `RGB2SH(torch.rand(...))` 把随机生成的 16 维向量先做一次 SH 变换（`scene/gaussian_model.py:139`），并存储为 `_objects_dc`（`scene/gaussian_model.py:158`）。你可以暂时把它当成"先随便给每个点发一张随机名牌，训练再慢慢纠正"即可，至于为什么走 SH 通道、与颜色 SH 的异同，留到数学篇细讲。

### 招式二：用 SAM 的 2D mask 当"伪标签"来监督（lifting 2D SAM）

那么问题来了：身份编码是 3D 的，我们手里却没有 3D 的物体标注（给 3D 点云逐点标物体太贵了）。怎么办？

作者巧妙地"借力"一个已经很强的 2D 分割模型——**SAM（Segment Anything Model）**，并配合 **DEVA**（Tracking-Anything-with-DEVA）把多张图里的 mask 关联起来。流程是：

1. 对**每一张训练照片**，让 SAM/DEVA 把图里的物体分割出来，得到一张张 2D 物体掩码（mask，每个像素标出它属于哪个物体实例）；
2. 这些 2D mask 不是人工标出来的真值，而是模型预测的，所以叫**伪标签（pseudo label）**；
3. 训练时，3D 高斯不仅被"泼"成一张 RGB 图，还额外被"泼"成一张**身份特征图（object feature map）**——也就是把每个高斯的 16 维身份编码投影到 2D 上；
4. 用一个轻量**分类器（classifier）**（一个 1×1 卷积，把 16 维特征映射成类别概率，见 `train.py:33`）把这张 16 通道特征图转成"每个像素属于哪个物体"的概率，再和 SAM 给的 2D mask 算交叉熵损失（`train.py:34`），反向传播去优化身份编码。

> 补充：分类器的输出通道数 `num_classes` 由配置文件指定，默认 **200**（`train.py:235`），代表一个场景最多同时区分 200 个物体实例。这个数要覆盖你数据集里 mask 的实例总数。

这一步的本质是：**把 2D 的物体分割"提升（lift）"到 3D**。我们不直接告诉 3D 点"你是谁"，而是告诉它"从某个角度看过去，你在 2D 上应该属于这个物体"，多张照片、多个视角一起约束，3D 的身份编码就被"逼"对了。

### 招式三：3D 空间一致性正则（spatial consistency regularization）

光靠 2D 伪标签还不够稳：照片总有看不到的角度、SAM 也难免出错。于是作者再加一道"保险"——**3D 空间一致性正则**。

直觉很朴素：**在 3D 空间里挨得很近的点，大概率属于同一个物体。** 比如椅子那一团点本来就挤在一块儿，它们的身份编码也应该一致。

具体做法（训练侧见 `train.py:97-101`，损失函数见 `utils/loss_utils.py:74` 的 `loss_cls_3d`）：

- 对每个 3D 点，用 **k 近邻（k-nearest neighbors, kNN）** 找到它最近的 $k$ 个邻居（默认 $k=5$，见 `train.py:237`）；
- 比较"这个点"和"它的邻居们"的身份概率分布，用 **KL 散度（KL divergence）** 衡量它们差得多远；
- 把这个差距作为一项损失加进去，逼着相邻点的身份趋于一致。

实现上还有一个工程小窍门：为了不让这一步拖慢训练，损失函数不会对全部点都算 KNN，而是**先按 `reg3d_max_points`（默认 300000）随机降采样、再按 `reg3d_sample_size`（默认 1000）随机抽一批点**来计算（见 `utils/loss_utils.py:88-98`）。此外它**不是每步都算**，而是每隔 `reg3d_interval`（默认 2）步才触发一次（`train.py:97`）。这些细节不影响理解，但能解释"为什么训练还能跑得动"。

简单说，这一招是"邻居之间别打架"——用 3D 几何的近邻关系，平滑、约束身份编码，让物体边界更干净。

### 三招合起来

最终的训练损失（在触发 3D 正则的步骤里）实际上是：

```math
\mathcal{L}=\underbrace{(1-\lambda_{\text{D-SSIM}})\,\mathcal{L}_{1}+\lambda_{\text{D-SSIM}}\,(1-\text{SSIM})}_{\text{RGB 重建：图像要像}}+\underbrace{\mathcal{L}_{\text{obj}}}_{\text{2D mask 监督身份}}+\underbrace{\mathcal{L}_{\text{3D}}}_{\text{空间一致性}}
```

其中 RGB 重建项同时用了 L1 和 D-SSIM（即 `1 - SSIM`），靠权重 $\lambda_{\text{D-SSIM}}$ 平衡，对应 `train.py:102`；不触发 3D 正则的步骤则去掉 $\mathcal{L}_{\text{3D}}$（`train.py:104`）。最终，**每个高斯既能渲染漂亮图片，又能告诉你它属于哪个物体**。这就是 Gaussian Grouping 的灵魂。

## 4. 能做什么（应用）

一旦高斯被"分组"，每个物体就是一组可以单独操控的点，编辑就变得非常灵活。对应 README 的演示，Gaussian Grouping 支持以下能力：

| 能力 | 说明 |
|---|---|
| **3D 重建（新视角合成）** | 继承 3DGS 的强项，渲染任意新视角的高质量图片 |
| **3D 开放世界分割** | 不限类别地分割场景里"任何东西"，并能用 grounded-SAM 给 mask 配上文字语义（见 `ext/grounded_sam.py`、`docs/dataset.md`） |
| **3D 物体移除** | 选一个物体 id，把它对应的点从场景里整组删掉（见 `edit_object_removal.py`、`docs/edit_removal_inpaint.md`） |
| **3D 物体修复（inpainting）** | 删掉物体后，把背后被遮挡、看不见的区域"补"出来，结合 2D 修复模型 LaMa + 微调高斯（见 `edit_object_inpaint.py`） |
| **风格迁移（style transfer）** | 给某个物体换一种艺术风格，多视角保持一致 |
| **多物体重组（recomposition）** | 把多个物体在 3D 里挪位置、重新组合 |

> 关键优势：因为高斯是**离散的、而且分了组**的，每个物体都能被"单独拎出来"，比 NeRF 那种"揉成一团、改一处影响全局"的隐式表示灵活得多。

---

## 5. 与 3DGS / NeRF 的关系

三者的对比如下：

| 维度 | NeRF | 3D Gaussian Splatting | **Gaussian Grouping** |
|---|---|---|---|
| 表示形式 | 隐式 MLP（神经网络） | 显式高斯点云 | 显式高斯点云 + **身份编码** |
| 渲染速度 | 慢（逐像素查询） | 实时 | 实时 |
| 是否能分割 | 较难（需额外设计） | 不支持 | **支持（开放世界分割）** |
| 是否方便编辑 | 困难（改一处影响全局） | 不支持 | **方便（按物体分组编辑）** |
| 物体级理解 | 无 | 无 | **有** |

一句话：Gaussian Grouping = 3DGS（高质量实时重建）+ 物体级身份（理解与编辑能力）。

几个子模块的角色再强调一下：

- **`submodules/diff-gaussian-rasterization`**：这是经过改造的 3DGS 光栅化器。原版只渲染 RGB；这里被改造成**多渲染一张"身份特征图（`render_object`，16 通道）"**，把 3D 身份编码泼到 2D，供分类器监督。训练时 `train.py:84` 正是从 `render_pkg["render_object"]` 取出这张图。这是把"身份"塞进可微渲染的关键。
- **`Tracking-Anything-with-DEVA`**：训练前的数据准备环节，负责对每张照片跑 SAM、并在多视角之间把"同一个物体"的 mask 关联起来，输出作为伪标签的 2D mask（见 `script/prepare_pseudo_label.sh`）。
- **`lama`**：物体修复时用的 2D 修复模型，负责把"删掉物体后露出的空洞"在图片层面先补上，再以此监督 3D 高斯微调。
- **`submodules/simple-knn`**：提供 k 近邻计算，3D 空间一致性正则（`loss_cls_3d`）依赖它。

## 7. 端到端流程（文字版流水线）

把整件事串起来，从你手里的照片到最终的编辑成品，大致经过这样一条流水线：

1. **原始照片**：你绕着场景拍的一组 2D 图片（放在 `input/` 下）。
2. **COLMAP 转换（`convert.py`）**：用 COLMAP 算出每张图的**相机位姿**（拍照时的位置和朝向）和一个**稀疏初始点云**。这是 3DGS 系列的标准起点。
3. **DEVA/SAM 生成伪标签（`script/prepare_pseudo_label.sh`）**：对每张照片分割出物体 mask，并在多视角间关联起来，得到训练用的 2D 物体伪标签。
4. **训练（`train.py`）**：以初始点云建 GaussianModel（每个点随机初始化 16 维身份编码），每轮迭代渲染 RGB 图 + 身份特征图，计算三项损失（RGB 重建、2D 身份、3D 一致性），反向传播优化；期间不断做**密度化（densification，哪里细节不够就在哪里再撒一把点）**和剪枝。产物：一组"既会画图又懂分组"的高斯 + 一个分类器。
5. **渲染（`render.py`）**：用训练好的模型，在任意新视角渲染出**漂亮图片**和**物体分割图**。
6. **编辑（`edit_object_*.py`）**：选定某个物体 id，做移除（直接删掉那组高斯）或修复（删掉后用 LaMa 补空洞、再微调高斯补全被遮挡区域）。产物：编辑后的 3D 场景。

每一步的产物，都为下一步提供输入，环环相扣。

## 8. 小结与阅读建议

- **Gaussian Grouping** 给每个 3D 高斯加了 16 维**身份编码**，让它不仅会"画图"，还懂"分物体"。
- 监督信号来自两处：**SAM 的 2D mask 伪标签**（把 2D 分割提升到 3D）+ **3D 空间一致性正则**（邻居应同属一物）。
- 因为高斯是**离散且分组**的，所以能轻松做移除、修复、风格迁移、重组等编辑，比 NeRF 灵活得多。

**建议阅读顺序**：

1. 本篇（建立整体直觉）；
2. **数学推导篇**（理解身份编码、密度化、KL 一致性背后的公式）；
3. **模型与渲染代码详解**（结合 `scene/gaussian_model.py`、`gaussian_renderer/__init__.py`、改造过的光栅化器）；
4. **训练与数据代码详解**（结合 `train.py`、数据读取、DEVA 伪标签生成）；
5. **编辑与评估代码详解**（结合 `edit_object_*.py`、LaMa、评估脚本）；
