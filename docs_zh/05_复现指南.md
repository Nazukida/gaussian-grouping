# 05 复现指南：从零跑通 Gaussian Grouping

> 本节你将学到：
> 1. 如何一步步配好 Gaussian Grouping 的 conda 环境，并编译两个 CUDA 子模块；
> 2. 怎么拿到（或自制）一份带 `object_mask` 的数据集；
> 3. 一条命令完成训练 + 分割渲染，并看懂每个参数；
> 4. 如何用 config 文件指定要移除 / 修复的物体并跑通编辑流程；
> 5. 新手最容易踩的 8 个坑，以及一份 5 行命令的"最小可行流程"。

本指南的"终点"很明确：**用 bear 数据集，从零跑通「训练 → 分割渲染 → 物体移除 → 评估」整条链路**。bear 是项目里最小、最快的数据集，最适合新手验证环境是否配通。

## 1. 硬件与系统要求

- **GPU**：必须是 NVIDIA 独立显卡。建议显存 **≥ 11 GB**（如 RTX 3060 12GB / 3080Ti / 4070 等），越大越快、越稳。bear 这种小数据集 8–11GB 也能跑，但大型场景（mipnerf360）最好 24GB。
- **CUDA 驱动**：本仓库依赖 PyTorch 1.12.1 + cudatoolkit=11.3，所以你的显卡驱动要支持 CUDA 11.3（驱动版本 ≥ 495 即可）。注意区分 **CUDA Toolkit（编译用）** 和 **驱动（运行用）**：编译子模块时需要本机装有 CUDA Toolkit（含 `nvcc`），且 Toolkit 大版本要和驱动匹配。
- **系统**：Linux / Windows 都行。本项目在 **Windows 10/11 也能跑通**，注意两点：① 路径建议用正斜杠 `/`；② 编译 CUDA 子模块需要 Visual Studio Build Tools + CMake。
- **编译工具链**：
  - Linux：`gcc` / `g++` / `cmake`（`sudo apt install build-essential cmake`）。
  - Windows：**Visual Studio 2019/2022 Build Tools**（勾选"使用 C++ 的桌面开发"）+ **CMake** + **CUDA Toolkit**，三者都要装且版本匹配。

> 比喻：把环境配好就像"搭好灶台"。灶台没搭稳，后面的菜（训练、渲染）再好也炒不出来。第 2、9 节会专门讲"灶台"问题。

## 2. 环境配置（核心，分小步）

以下命令与 `docs/install.md:1-21` 完全一致。

### 第 1 步：克隆仓库

```bash
git clone https://github.com/lkeab/gaussian-grouping.git
cd gaussian-grouping
```

> 注意：本仓库**没有使用 git submodule**（根目录没有 `.gitmodules`）。`submodules/` 下的两个目录（`diff-gaussian-rasterization`、`simple-knn`）是**直接作为普通文件提交在主仓库里**的，`git clone` 时会随主仓库一起下载下来，**不需要** `--recursive`，也**不需要** `git submodule update`。

**关键**：`submodules/diff-gaussian-rasterization` 是**作者改过的版本**（在 rasterizer 里增加了 `sh_objs` 输入与 `objects` 输出，用来渲染身份特征），**不能用官方原版替换**，否则渲染时会报 `render() got an unexpected keyword argument 'sh_objs'`。

下载完成后，确认这两个目录都**不是空的**（能看到 `.py` / `.cu` 源文件即可）：

```bash
# 期望：两个目录都列出文件，而非空
ls submodules/diff-gaussian-rasterization/diff_gaussian_rasterization/
ls submodules/simple-knn/
```

如果某个目录是空的（极少数情况下下载不完整），重新 `git clone` 一次即可，**不要**去找 submodule 命令——它们不是 submodule。

### 第 2 步：创建 conda 环境

```bash
conda create -n gaussian_grouping python=3.8 -y
conda activate gaussian_grouping
```

注意 Python 版本严格用 **3.8**。PyTorch 1.12.1 + 这个 rasterizer 在 3.8 上最稳。

### 第 3 步：安装 PyTorch（带 CUDA 11.3）

```bash
conda install pytorch==1.12.1 torchvision==0.13.1 torchaudio==0.12.1 cudatoolkit=11.3 -c pytorch
```

装完进 Python 验一下：

```bash
python -c "import torch; print(torch.__version__, torch.cuda.is_available())"
# 期望输出类似：1.12.1+cu113 True
```

如果第二项是 `False`，说明 CUDA 没接上，先解决驱动 / Toolkit 问题再继续。

### 第 4 步：用 pip 装其余依赖

```bash
pip install plyfile==0.8.1
pip install tqdm scipy wandb opencv-python scikit-learn lpips
```

`plyfile` 用来读写 `.ply`（高斯点云的存储格式）；`lpips` 是感知损失用的；其余是常规工具库。

### 第 5 步：编译两个 CUDA 子模块（最容易翻车的一步）

```bash
pip install submodules/diff-gaussian-rasterization
pip install submodules/simple-knn
```

这一步会调用 `nvcc` + 你的 C++ 编译器，从源码编译成 `.pyd` / `.so`。编译时间从 1 分钟到十几分钟不等。

**编译失败的常见原因（详见第 9 节 FAQ）**：

1. **CUDA Toolkit 与显卡驱动不匹配**：例如驱动只支持到 11.2，却装了 11.6 的 Toolkit。
2. **Windows 缺 Visual Studio Build Tools**：报错里出现 `error: command 'cl.exe' failed` 或 `Microsoft Visual C++ 14.0 is required`。
3. **`nvcc` 找不到**：PATH 里没有 CUDA 的 `bin` 目录，命令行里敲 `nvcc --version` 无输出。
4. **官方原版 diff-gaussian-rasterization 冲突**：之前 pip 装过原版，现在 `import` 进来的是旧版，渲染时报 `render() got an unexpected keyword argument 'sh_objs'`。
5. **装错成官方原版 diff-gaussian-rasterization**：之前用 `pip install` 装过原版，`import` 进来的是旧版。务必先 `pip uninstall diff-gaussian-rasterization`，再 `pip install submodules/diff-gaussian-rasterization`（本仓库自带版本）。

编译成功后验证：

```bash
python -c "from diff_gaussian_rasterization import GaussianRasterizer; print('ok')"
```

## 4. 数据集获取（两条路）

### 路 A（推荐新手）：直接下预转换数据集

到 HuggingFace 下载：`https://huggingface.co/mqye/Gaussian-Grouping/tree/main` 的 `data/` 目录，里面已经包含 `bear`、`lerf`、`mipnerf360`，且都带好了 `object_mask`。

下载后放到项目根目录，目录结构应为（以 bear 为例）：

```
data/bear/
├── images/                 # 原图（去畸变后）
│   ├── 00000.png
│   ├── 00001.png
│   └── ...
├── object_mask/            # 每张图对应的"物体身份灰度图"（伪标签）
│   ├── 00000.png
│   └── ...
└── sparse/0/               # COLMAP 重建结果
    ├── cameras.bin         # 相机内参
    ├── images.bin          # 相机外参（位姿）
    └── points3D.bin        # 初始稀疏点云
```

> 关键概念：`object_mask` 里每个像素的灰度值 = 这个像素所属物体的"身份编号（id）"。训练时模型就是学着把这些 id 复制到每个 3D 高斯上（即"身份编码/Identity Encoding"）。

### 路 B（自制数据集）

如果你有自己的照片，按 `docs/train.md:18-49` 走：

**① 准备原始照片**

```
<location>/
└── input/
    ├── IMG_0001.jpg
    ├── IMG_0002.jpg
    └── ...
```

**② 跑 COLMAP 重建相机位姿和初始点云**

```bash
python convert.py -s <location>
```

`convert.py:31-78` 会依次调用 COLMAP 的：特征提取（`convert.py:35-44`）→ 特征匹配（`convert.py:47-53`）→ bundle adjustment / Mapper（`convert.py:58-66`）→ 去畸变（`convert.py:70-78`），最终在 `<location>/` 下生成 `sparse/0/`（含三个 `.bin`）和去畸变后的 `images/`。

> 前提：本机已装 **COLMAP** 并在 PATH 中（命令行敲 `colmap` 有反应）。否则会报 `Feature extraction failed`。`convert.py` 还支持 `--camera OPENCV`、`--resize`（生成 `images_2/4/8`，需 ImageMagick）等参数。

**③ 用 DEVA 生成物体伪标签**

```bash
# 例 1：bear 数据集
bash script/prepare_pseudo_label.sh bear 1

# 例 2：figurines（lerf 子目录）
bash script/prepare_pseudo_label.sh lerf/figurines 1

# 例 3：counter（mipnerf360 子目录）
bash script/prepare_pseudo_label.sh mipnerf360/counter 2
```

`script/prepare_pseudo_label.sh` 会进入 `Tracking-Anything-with-DEVA/`，跑两次 `demo/demo_automatic.py`：一次出彩色可视化 mask（`script/prepare_pseudo_label.sh:32-43`），一次出灰度训练 mask（带 `--use_short_id`，`script/prepare_pseudo_label.sh:46-56`），最后把灰度 mask 复制到 `data/<name>/object_mask/`（`script/prepare_pseudo_label.sh:58`）。

**`scale` 参数含义**（`script/prepare_pseudo_label.sh:25-29`）：

| scale | 读取的图片目录 | 说明 |
| --- | --- | --- |
| 1 | `images` | 用原图 |
| 2 | `images_2` | 用 1/2 分辨率图 |
| 4 | `images_4` | 用 1/4 分辨率图 |

> 比喻：mask 质量直接决定 3D 分割效果。DEVA 出 mask 很快，建议先调好"任意物体分割"参数（脚本里默认 `--SAM_PRED_IOU_THRESHOLD 0.7`、`--suppress_small_objects`），让多视图的 mask 尽量一致、合理，再正式训练。

## 5. 训练

### 命令（bear 数据集）

```bash
bash script/train.sh bear 1
```

等价于（见 `script/train.sh:21-24`）下面这两条：

```bash
# 1) 训练
python train.py -s data/bear -r 1 -m output/bear --config_file config/gaussian_dataset/train.json

# 2) 用训好的模型做分割渲染
python render.py -m output/bear --num_classes 256
```

### 参数逐个解释

| 参数 | 含义 |
| --- | --- |
| `-s data/bear` | 源数据路径（source），里面要有 `images/`、`object_mask/`、`sparse/0/` |
| `-r 1` | 分辨率缩放（resolution）。`1` = 原图；`2` = 1/2 分辨率；`4` = 1/4。显存不够就调大这个数 |
| `-m output/bear` | 模型输出目录（model path），权重、渲染结果都存这里 |
| `--config_file config/gaussian_dataset/train.json` | 超参文件（见下表） |
| `--train_split` | **仅 LERF 类数据集需要**（`script/train_lerf.sh:21`）。LERF 用 `images_train/` 划分训练集，而不是默认的全部 `images/`。bear 不用加；要训 lerf 请改用 `bash script/train_lerf.sh <name> <scale>` |

### `config/gaussian_dataset/train.json` 字段（`config/gaussian_dataset/train.json:1-9`）

| 字段 | 值 | 含义 |
| --- | --- | --- |
| `densify_until_iter` | 10000 | 在前 10000 步做密度化（densification），即"哪里细节不够就在哪里再撒一把高斯" |
| `num_classes` | 256 | 身份类别数（最大可表达的物体 id + 1） |
| `reg3d_interval` | 5 | 每 5 步做一次 3D 正则化（身份一致性约束） |
| `reg3d_k` | 5 | 正则化时每个点的近邻数 |
| `reg3d_lambda_val` | 2 | 正则化权重 |
| `reg3d_max_points` | 200000 | 正则化采样的最大点数（显存吃紧可调小） |
| `reg3d_sample_size` | 1000 | 每次正则化采样的 batch 大小 |

### 训练时长与保存

- **总迭代数**：30000 步（默认 `opt.iterations = 30_000`，见 `arguments/__init__.py:75`）。
- **保存点**：`save_iterations` 默认是 `[1_000, 7_000, 30_000, 60_000]`（`train.py:212`），代码还会再追加一个 `args.iterations`（即 30000，见 `train.py:221`）。由于训练只跑到 30000 步，**60_000 实际不会触发**，因此真正落到磁盘的检查点是 1000、7000、30000 三处（每个目录里都含 `point_cloud.ply` 和 `classifier.pth`）。保存路径形如：
  ```
  output/bear/point_cloud/iteration_7000/{point_cloud.ply, classifier.pth}
  output/bear/point_cloud/iteration_30000/{point_cloud.ply, classifier.pth}
  ```
- **时长**：bear 在 RTX 30/40 系显卡上大约 10–20 分钟；大型场景可能要数小时。
- **用 wandb 记录 loss**：加 `--use_wandb`（`train.py:218`），首次使用前先 `wandb login`。

## 6. 分割渲染

训练脚本已经自动接了渲染，也可以单独跑：

```bash
python render.py -m output/bear --num_classes 256
```

- `--num_classes 256`：与训练时的 `num_classes` 保持一致。
- `--skip_train` / `--skip_test`：跳过训练集 / 测试集的渲染（`render.py:159-160`）。

输出目录（`render.py:77-81`，`iteration` 取实际加载的 `loaded_iter`，一般是 30000）：

```
output/bear/train/ours_30000/
├── renders/            # 渲染出的 RGB 图
├── gt/                 # 真实图（ground truth）
├── objects_feature16/  # 每个 high-dim 身份特征图的渲染（16 维）
├── gt_objects_color/   # 真实的彩色物体 mask
├── objects_pred/       # 模型预测的"每个像素属于哪个物体"的彩色图  ← 选 id 时看这里
└── concat/
    └── result.mp4      # 拼接视频，5 列横向并排
```

> `result.mp4`（`render.py:109-129`）是把 5 张图横向拼成一帧录制下来的：**真实图 | 渲染图 | 真实 mask | 预测 mask | 特征图**（`render.py:123` 的 `np.hstack` 一共拼了 5 个）。这就是新手最想看到的"成果"：相机转一圈，每个物体被涂上不同颜色。

## 7. 编辑

### 7.1 物体移除（Object Removal）

```bash
bash script/edit_object_removal.sh output/bear config/object_removal/bear.json
```

等价于（`script/edit_object_removal.sh:21`）：

```bash
python edit_object_removal.py -m output/bear --config_file config/object_removal/bear.json
```

`config/object_removal/bear.json`（`config/object_removal/bear.json:1-4`）：

| 字段 | 值 | 含义 |
| --- | --- | --- |
| `num_classes` | 256 | 同训练 |
| `removal_thresh` | 0.3 | 概率阈值。某高斯属于目标物体的概率 > 0.3 就删掉（`edit_object_removal.py:76`） |
| `select_obj_id` | `[34]` | 要移除的物体 id 列表（可以多个） |

> 比喻：移除不是只删"贴着熊"的那几个点。代码先用概率阈值选出一批点，再用这些点算一个"凸包（convex hull）"，把凸包内部的所有高斯也一起删掉（`edit_object_removal.py:79-80`）——就像把熊的轮廓连同它"挡住的内部空间"整块挖掉，避免剩下空壳。

**怎么确定 `select_obj_id`？**

1. 先 `render.py` 渲染一遍，打开 `output/bear/train/ours_30000/objects_pred/` 里的彩色图，找到你想删的物体（比如那只熊）对应的颜色。
2. 项目用一个固定的 `id2rgb` 映射把 id 映射成颜色（`render.py:47-65`）。可以写个小脚本反查：拿到该颜色 → 反推 id。bear 里那只熊的 id 正好是 **34**，所以 config 默认就填 34。
3. 多个物体一起删就写成列表，如 `[34, 12]`。

输出：被删物体后的新点云存到（`edit_object_removal.py:89`）：

```
output/bear/point_cloud_object_removal/iteration_30000/point_cloud.ply
```

### 7.2 物体修复（Object Inpaint）

完整流程是三步（见 `docs/edit_removal_inpaint.md:28-34`）：

1. **移除物体**（即 7.1）。
2. **(可选)** 用 DEVA 找到移除后留下的"看不见区域 mask（unseen mask）"，再用 **LaMa** 做 2D 修复，得到伪标签。命令见 `docs/edit_removal_inpaint.md:40-85`。
3. **用 2D 修复结果作为伪标签，微调（finetune）3D 高斯**。

**新手请直接跳到第 3 步**：bear 数据集在 HuggingFace 上已经提供了伪标签（`images_inpaint_unseen/` 和 `inpaint_object_mask_255/`），前两步可以全部跳过。

第 3 步命令：

```bash
bash script/edit_object_inpaint.sh output/bear config/object_inpaint/bear.json
```

等价于（`script/edit_object_inpaint.sh:21`）：

```bash
python edit_object_inpaint.py -m output/bear --config_file config/object_inpaint/bear.json
```

`config/object_inpaint/bear.json`（`config/object_inpaint/bear.json:1-9`）相比 removal 多了几个字段：

| 字段 | 值 | 含义 |
| --- | --- | --- |
| `images` | `"images_inpaint_unseen"` | 用 LaMa 修复后的图作为微调输入 |
| `object_path` | `"inpaint_object_mask_255"` | 修复区域的二值 mask（255 = 修复区） |
| `r` | 1 | 分辨率缩放 |
| `lambda_dlpips` | 0.5 | 感知损失（LPIPS）权重，让修复区看起来自然 |
| `finetune_iteration` | 10000 | 微调步数 |

> 小贴士：在代码里 `lambda_dlpips` 实际被读进 `args.lambda_dssim`（`edit_object_inpaint.py:239`），微调时控制渲染图与修复图之间的 LPIPS 损失权重。微调只放开修复区附近的高斯参数，其余点固定，所以比从零训练快得多。

修复后的点云存到（`edit_object_inpaint.py:116`）：

```
output/bear/point_cloud_object_inpaint/iteration_9999/point_cloud.ply
```

## 8. 评估（LERF-Mask 数据集）

这一步**针对 LERF-Mask 数据集**（figurines / ramen / teatime），不是 bear。流程见 `docs/dataset.md`：

**① 下载数据**：从 HuggingFace 的 `data/lerf_mask` 下载，结构含 `test_mask/`（每个 novel view 下有按"文本提示"命名的 mask）。

**② 用文本提示渲染预测 mask**（需要第 3 节的 Grounded-SAM 环境 + HuggingFace 上的 checkpoint）：

```bash
python render_lerf_mask.py -m output/lerf_pretrain/figurines --skip_train
```

`render_lerf_mask.py:114-121` 内置了三个数据集的文本提示词（figurines / ramen / teatime），其他场景会直接 `raise NotImplementedError`。它会在第一帧用 Grounded-SAM 根据文本框出物体、对齐到模型的物体 id（`render_lerf_mask.py:41-50`），再渲染出每个视角的二值预测 mask。

**③ 算 IoU / Boundary-IoU**：

```bash
python script/eval_lerf_mask.py figurines
```

> 注意路径对齐（最容易卡住的一步）：`eval_lerf_mask.py:9-11` 把预测路径写死成 `result/lerf_mask/<dataset>`，而第②步渲染出的结果在 `output/lerf_pretrain/<dataset>/test/ours_<iter>_text/test_mask/` 下，二者不一致。更要命的是**目录层级对不上**：评估脚本按 GT 的 `data/lerf_mask/<dataset>/test_mask/<novel视图名>/<prompt>.png` 两级结构去 `result/lerf_mask/<dataset>/<novel视图名>/<prompt>.png` 找预测，但渲染产出是 `test_mask/<整数idx>/<prompt>.png`——第一级是整数序号而非视图名。**简单整体移动/软链接会导致一级目录名对不上，IoU 全部取不到。** 建议写个小脚本，按 GT `test_mask/` 下的子目录名（视图顺序）把渲染的 `test_mask/<idx>/` 重命名对应过去，或直接改 `eval_lerf_mask.py:11` 的 `pred_folder_path` 指到正确结构。三个数据集同理把 `figurines` 换成 `ramen` / `teatime`。

## 9. 常见坑（FAQ，新手最需要）

| # | 现象 | 原因 / 解决 |
| --- | --- | --- |
| 1 | `ModuleNotFoundError: No module named 'diff_gaussian_rasterization'` | 子模块没编译安装。本仓库不用 git submodule，`submodules/` 随主仓库下载；确认目录非空后，执行 `pip install submodules/diff-gaussian-rasterization` 与 `pip install submodules/simple-knn` |
| 2 | `render() got an unexpected keyword argument 'sh_objs'` | 装了**官方原版** diff-gaussian-rasterization。卸载后重装本仓库自带版本：`pip uninstall diff-gaussian-rasterization` → `pip install submodules/diff-gaussian-rasterization` |
| 3 | Windows 上 `cl.exe failed` / `nvcc not found` | 没装 VS Build Tools 或 CUDA Toolkit 不在 PATH。装 VS 2019/2022（勾 C++ 桌面开发），把 CUDA 的 `bin` 加进系统 PATH，重开终端 |
| 4 | `convert.py` 报 `Feature extraction failed` | COLMAP 没装或不在 PATH。到 [colmap.github.io](https://colmap.github.io/) 装好，命令行敲 `colmap -h` 能出帮助即 OK |
| 5 | 训练时报某个物体 id 越界 / 分割全错 | `num_classes` 必须 **大于** mask 中最大的物体 id。DEVA 默认 id 可能很大，所以项目统一用 **256**。自制数据时若 id 超过 256，要么调大 `num_classes`，要么在 DEVA 里压缩 id |
| 6 | `CUDA out of memory` | ① 调大 `-r`（如 `-r 2`）；② 调小 `config` 里的 `reg3d_max_points`（如 100000）；③ 换更大显存的卡 |
| 7 | 编辑时报 `Delaunay ... degenerate` / convex hull 报错 | 点云过度共面导致 Delaunay 三角化退化。多半是某个物体点太少或全在一个平面，换个 `select_obj_id` 或微调 `removal_thresh` |
| 8 | 加载数据时崩溃（`torch.from_numpy` 报错 / `None`） | `object_mask/` 缺文件。mask 固定为 `<图片主名>.png`（即使原图是 `.jpg`，mask 也是 `.png`，见 `scene/dataset_readers.py:96-99`）。某张图缺 mask 会让 `objects=None`，进而在 `utils/camera_utils.py:54` 的 `np.array(None)` 处直接崩溃——不是"分割全黑"，而是启动即报错 |

> 一个小技巧：每次报错先看**第一行**报错信息（不是 traceback 最后一行），它通常直接点明问题（如 `FileNotFoundError` → 路径错了）。

## 10. 最小可行流程（TL;DR）

如果你已经配好环境（第 2 节）、下载了 bear 数据集（路 A），那么从零到看到分割视频只要 5 行：

```bash
# 0. 激活环境
conda activate gaussian_grouping

# 1. 训练（约 10-20 分钟）+ 自动渲染分割结果
bash script/train.sh bear 1

# 2. 看分割视频（在资源管理器里双击即可）
#    output/bear/train/ours_30000/concat/result.mp4

# 3. 移除 id=34 的物体（那只熊）
bash script/edit_object_removal.sh output/bear config/object_removal/bear.json

# 4. （可选）用预置伪标签做修复
bash script/edit_object_inpaint.sh output/bear config/object_inpaint/bear.json
```

跑通这 5 行，恭喜你——你已经完整复现了 Gaussian Grouping 的核心功能。

## 小结

- 环境配置的关键是 **Python 3.8 + PyTorch 1.12.1 + CUDA 11.3 + 本仓库自带的两个 CUDA 子模块**，子模块一定不能少、不能用官方版替换。
- 数据有两条路：新手直接下 HuggingFace 预转换数据（带 `object_mask`）；自制数据要 `convert.py`（COLMAP）+ `script/prepare_pseudo_label.sh`（DEVA）。
- 训练入口是 `script/train.sh <name> <scale>`（LERF 类用 `train_lerf.sh`），等价于 `train.py` + `render.py` 两条命令；超参在 `config/gaussian_dataset/train.json`。训练 30000 步，检查点落在 1000 / 7000 / 30000。
- 编辑靠 config 文件指定 `select_obj_id`：移除用 `object_removal/<name>.json`，修复用 `object_inpaint/<name>.json`；移除会按概率阈值 + 凸包挖空，修复靠 LaMa 伪标签微调。
- LERF-Mask 评估面向 figurines/ramen/teatime，不是 bear；需要 Grounded-SAM 环境，且要手动对齐预测目录到 `result/lerf_mask/`。
- 报错先查第 9 节 FAQ，绝大多数新手问题（子模块、官方版冲突、COLMAP、显存、id 越界）都能在那里找到答案。
