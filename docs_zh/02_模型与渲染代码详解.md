## 1. GaussianModel 是什么

在 PyTorch 的世界里，我们习惯了用 `nn.Module` 搭建神经网络：一堆卷积层、激活函数串起来，输入一张图，输出一个预测。但 3D 高斯泼溅（3D Gaussian Splatting, 3DGS）的思路完全不同。

看 `scene/gaussian_model.py:22`：

```python
class GaussianModel:
```

注意它**不是** `nn.Module` 的子类，而是一个普通类。它内部用 `nn.Parameter` 存的不是"权重"，而是**一堆 3D 高斯点本身的所有属性**——位置、颜色、大小、朝向、透明度，外加本项目独有的"身份"。

打个比方：`GaussianModel` 就像一个**装着几百万颗半透明、可变形、会发光的水母的大水族箱**。每颗水母有自己固定的一组参数（中心在哪、多大、朝哪转、多透明、发什么光），训练的目标就是反复微调这些参数，让从任意角度看过去，这群水母叠在一起的样子都和真实照片一致。

既然它不是 `nn.Module`，那梯度从哪来？答案是：所有 `_xxx` 张量都注册成了 `nn.Parameter(...requires_grad_(True))`（如 `scene/gaussian_model.py:151-158`），它们能被一个普通的 `torch.optim.Adam` 优化器更新；而可微的"渲染"由自定义 CUDA 光栅化器完成（见第 8、9 节）。

## 2. 每颗高斯的属性

把 `GaussianModel.__init__`（`scene/gaussian_model.py:42`）和 `create_from_pcd`（`:130`）放一起看，每颗高斯一共携带 7 组参数（设共有 $N$ 颗）：

| 属性变量 | 形状 | 物理含义 | 取用时的激活函数 |
|---|---|---|---|
| `_xyz` | $(N,3)$ | 高斯中心在世界坐标的位置 | 无（直接用） |
| `_features_dc` | $(N,1,3)$ | 球谐 DC 系数（视角无关的基础颜色） | 后接 `+0.5`、`clamp_min` |
| `_features_rest` | $(N,15,3)$ | 球谐高阶系数（视角相关颜色） | 同上 |
| `_scaling` | $(N,3)$ | 三个轴的**对数**缩放 | `torch.exp` |
| `_rotation` | $(N,4)$ | 四元数表示朝向 | `normalize` |
| `_opacity` | $(N,1)$ | 不透明度的 logit | `sigmoid` |
| `_objects_dc` | $(N,1,16)$ | **身份编码 Identity Encoding**（本项目新增） | 与 DC 类似处理 |

### 2.1 `_xyz`：位置/中心

最直观的属性。每颗高斯是一个 3D 椭球，它的"中心点"就是 `_xyz` 里的那个 $(x,y,z)$。

### 2.2 `_features_dc` 与 `_features_rest`：球谐系数存颜色

3DGS 用球谐函数（Spherical Harmonics, SH）来表示**视角相关的颜色**——同一颗高斯，从正面看可能是亮的，从侧面看可能是暗的，像金属表面的反光。

设最大球谐阶数 `max_sh_degree = 3`，则每个颜色通道的系数总数为

```math
(\text{max\_sh\_degree}+1)^2 = (3+1)^2 = 16
```

这 16 个系数被拆成两部分：
- **DC 项（直流分量）** `f_dc`：阶数 0，只有 1 个系数，代表"平均颜色"，形状为 `_features_dc` 的 $(N,1,3)$（"1" 是 1 个 SH 系数，"3" 是 RGB）。
- **高阶项** `f_rest`：剩下 15 个系数，描述颜色随方向变化的部分，形状为 $(N,15,3)$。

初始化时 `features` 的形状是 $(N,3,16)$，其中通道维 `3` 是 RGB 三色，每个颜色都有自己独立的 16 个 SH 系数。代码 `features[:, :3, 0] = fused_color` 存 DC，`features[:, :, 1:] = 0.0` 把高阶置零（`scene/gaussian_model.py:134-136`）。随后分别 `transpose(1,2)` 整理成上面表中的存储形状（`:152-153`）。注意 SH 的 DC 系数并不是直接的 RGB 值，需要经过 `SH2RGB` 变换（见第 10 节）。

### 2.3 `_scaling`：对数缩放，经 `exp` 激活

高斯椭球在三个主轴上的"半径"。但代码存的是**对数缩放**，取用时 `torch.exp`（`scene/gaussian_model.py:99`）。

> **为什么存对数？** 因为真实的缩放必须 $>0$（半径不可能为负）。如果直接优化缩放本身，优化器可能把它推到负数，导致协方差矩阵不再正定、高斯"塌掉"。存 $\log s$ 后，无论梯度怎么走，$s=\exp(\log s)$ 永远是正的。这就把"必须为正"这个硬约束自动满足了，而且数值稳定。这是一个无约束优化的小技巧。

### 2.4 `_rotation`：四元数，`normalize` 后表示朝向

椭球怎么转，用四元数 $(w,x,y,z)$ 表示，形状 $(N,4)$。四元数必须是单位长度才对应一个有效的旋转，所以取用时做 `torch.nn.functional.normalize`（`scene/gaussian_model.py:103`）。

> 同样的思路：直接优化四元数，再在用的时候归一化，避免在优化空间里硬加"模长为 1"的约束。

### 2.5 `_opacity`：经 `sigmoid` 得到不透明度

每颗高斯有多不透明，存在 $(N,1)$ 里，取用时 `sigmoid`（`scene/gaussian_model.py:121`）。

> **为什么存 logit？** 不透明度 $\alpha \in (0,1)$。如果直接优化 $\alpha$，需要剪裁到 $[0,1]$，梯度在边界处会失效。存它的 logit（即 `inverse_sigmoid`，见 `utils/general_utils.py:18` 的 $\log(x/(1-x))$）后，sigmoid 会自动把任意实数映射回 $(0,1)$，又是无约束优化。

### 2.6 `_objects_dc`：身份编码（本项目灵魂）

这是 Gaussian-Grouping 相对原版 3DGS 最关键的新增。形状是 $(N,1,16)$，其中 `16` 对应 `self.num_objects = 16`（`scene/gaussian_model.py:52`）。

理解它的形状最简单的办法：把它和 DC 颜色对比。DC 颜色是 $(N,1,3)$——每颗高斯带一个长度为 3 的向量（RGB）作为它的"基础属性"。身份编码则是把 3 换成 16，每颗高斯带一个长度为 16 的向量作为它的"身份"。**它就像一个 16 维的"身份证号"**，渲染出来后会变成一张 16 通道的身份特征图（object feature map），再用一个分类器（classifier）把它变成像素级的物体类别。

为什么用"DC 的存法"？因为渲染时身份特征走的管线和颜色**完全一样**（都是 0 阶 SH，即一个常数向量），复用同一套 alpha 合成代码最省事。第 9 节会看到光栅化器是怎么做到的。

## 3. setup_functions：激活函数与协方差构造

`scene/gaussian_model.py:24` 的 `setup_functions` 把上面提到的激活函数集中注册：

```python
self.scaling_activation = torch.exp
self.scaling_inverse_activation = torch.log
self.covariance_activation = build_covariance_from_scaling_rotation
self.opacity_activation = torch.sigmoid
self.inverse_opacity_activation = inverse_sigmoid
self.rotation_activation = torch.nn.functional.normalize
```

（`scene/gaussian_model.py:31-39`）三个激活的意义已经在第 2 节解释：让优化器在无约束的实数空间里自由走动，取用时再变回物理上有意义的值。

还有一个 `build_covariance_from_scaling_rotation`（`:25`）：高斯的形状除了大小（scaling）还和朝向（rotation）有关，二者合起来构成了一个 $3\times3$ 的**协方差矩阵** $\Sigma$，它决定了椭球的精确形状。代码里先构造 $L = R\cdot\text{diag}(s)$，再算 $\Sigma = L L^\top$，最后只保留对称的 6 个独立分量（因为 $\Sigma$ 对称，用 `strip_symmetric` 抽取，`utils/general_utils.py:75`）。

```math
\Sigma = L L^\top,\qquad L = R\,\text{diag}(s)
```

这里 `R` 由四元数经 `build_rotation` 得到（见第 10 节），`s` 是激活后的缩放。详细的几何含义和 EWA splatting 投影推导我们留到数学篇展开，这里只需知道：**协方差由 scaling + rotation 自动算出来，光栅化器据此把每颗高斯投影成屏幕上的 2D 椭圆斑。**

## 4. create_from_pcd：从 SfM 点云初始化

训练前，3DGS 需要"撒一把初始的高斯点"。这些点通常来自 SfM（Structure from Motion）重建出的稀疏点云。`scene/gaussian_model.py:130` 的 `create_from_pcd` 完成初始化。

### 4.1 位置与颜色

```python
fused_point_cloud = torch.tensor(np.asarray(pcd.points)).float().cuda()   # (N,3)
fused_color = RGB2SH(torch.tensor(np.asarray(pcd.colors)).float().cuda()) # RGB → SH 空间
```

点云自带颜色（RGB），但 3DGS 存的是 SH 系数，所以先用 `RGB2SH` 把 RGB 搬到 SH 空间（`utils/sh_utils.py:114`）：

```math
\text{SH}_{dc} = \frac{\text{RGB} - 0.5}{C_0},\qquad C_0 \approx 0.28209479
```

DC 系数填进 `features[:, :3, 0]`，高阶位置全部置零（`:134-136`），即初始时刻每颗高斯只有"平均颜色"，没有视角相关变化。

### 4.2 身份编码的随机初始化（本项目改动）

```python
fused_objects = RGB2SH(torch.rand((fused_point_cloud.shape[0], self.num_objects), device="cuda"))  # :139
fused_objects = fused_objects[:, :, None]   # (N,16,1)
```

注意这里又用了一次 `RGB2SH`，但作用对象是 `torch.rand` 出来的随机数（范围 $[0,1)$）。为什么要套一层 `RGB2SH`？

> 因为后面渲染时，身份特征要经过和颜色**完全相同**的反变换（`SH2RGB`，即 `sh \cdot C_0 + 0.5`）才能进入 alpha 合成。如果初始化时存的是裸随机数，渲染时再 `+0.5` 就会偏移到 $[0.5, 1.5)$，整体偏亮。先用 `RGB2SH`（减 0.5 除 $C_0$）把随机值"搬到 SH 空间"，渲染时再变回来，数值范围才一致。本质上是**让身份特征的存储约定和 DC 颜色对齐**。

之后 `transpose(1,2)` 把形状整理成 $(N,1,16)$（`:158`），与 DC 颜色的存储方式一致。

### 4.3 缩放、旋转、不透明度

- **缩放**：用 `simple_knn` 的 `distCUDA2` 算每个点到最近邻的平方距离 $d^2$（并 `clamp_min` 到 $10^{-7}$ 防止除零），再 `log(sqrt(d^2))` 作为三个轴的对数缩放（`:144-145`）。直觉上：点稀疏的地方邻居远、$d^2$ 大、初始高斯就大；点密集的地方初始高斯就小。这样初始覆盖就比较合理。
- **旋转**：单位四元数 $(1,0,0,0)$，即"不转"（`:146-147`）。
- **不透明度**：`inverse_sigmoid(0.1)`（`:149`）。即初始每颗高斯的真实不透明度是 $0.1$，比较透明，让一开始叠出来的图不至于太"实"，给优化留余地。

## 5. training_setup：7 组参数与学习率

`scene/gaussian_model.py:160` 把 7 组参数交给一个 `Adam` 优化器，并给每组配学习率（`:165-173`）：

| 参数名 (`name`) | 张量 | 学习率 |
|---|---|---|
| `xyz` | `_xyz` | `position_lr_init * spatial_lr_scale`（指数衰减调度） |
| `f_dc` | `_features_dc` | `feature_lr` |
| `f_rest` | `_features_rest` | `feature_lr / 20.0` |
| `opacity` | `_opacity` | `opacity_lr` |
| `scaling` | `_scaling` | `scaling_lr` |
| `rotation` | `_rotation` | `rotation_lr` |
| `obj_dc` | `_objects_dc` | `feature_lr` |

几个要点：

1. **`obj_dc` 复用 `feature_lr`**。身份编码和颜色 DC 同等重要，所以共享颜色的学习率。
2. **高阶 SH 学习率除以 20**。高阶项是"细节修正"，初始为 0，步子迈太大容易抖。
3. **位置 `xyz` 用指数衰减调度**。`get_expon_lr_func`（`utils/general_utils.py:29`）在第 0 步给 `lr_init`、在 `max_steps` 步给 `lr_final`，中间用对数空间线性插值（等价于实际空间的指数衰减）；开头还有一段 `lr_delay_mult` 的"暖身"。`update_learning_rate`（`scene/gaussian_model.py:349`）每步调用它来设置 `xyz` 的当前学习率。
4. **`spatial_lr_scale` 与场景尺寸挂钩**。大场景里点之间距离远，位置学习率要按比例放大，所以 `xyz` 的学习率乘上了这个缩放因子（在 `create_from_pcd` 里由外部传入）。

## 6. 密度化 densification：哪里不够补哪里

这是 3DGS 训练中最"活"的部分，也是新手最容易看不懂的地方。

### 6.1 为什么需要密度化

初始点云通常只有几万到几十万个点，远不够还原所有细节。训练中会发现：有些地方点太稀疏、重建不出来；有些高斯被优化得越来越大、糊成一片；还有些点根本看不见、白白占内存。

> 比喻：密度化就像"哪里细节不够就在哪里再撒一把点，太大的水母要拆成两颗更小的，看不见的水母直接捞走"。

### 6.2 累积屏幕空间梯度：`add_densification_stats`

```python
def add_densification_stats(self, viewspace_point_tensor, update_filter):  # :598
    self.xyz_gradient_accum[update_filter] += torch.norm(
        viewspace_point_tensor.grad[update_filter, :2], dim=-1, keepdim=True)
    self.denom[update_filter] += 1
```

每次渲染后，把每个高斯在**屏幕空间**（2D）位置上的梯度累加起来。直觉：如果某颗高斯在屏幕上的位置怎么放都不对（梯度大），说明这里"信息不够"，需要更多点来表达。`denom` 是累计的被观察次数，用来求平均。

### 6.3 clone 与 split

`densify_and_clone`（`:566`）和 `densify_and_split`（`:540`）分别处理两种情况，判据都由两部分组成：**屏幕梯度大于阈值** 且 **大小满足某条件**。

- **clone（复制）**：点小但梯度大——说明这颗小高斯被反复"拉扯"，需要复制一份。
  ```python
  selected_pts_mask = torch.norm(grads, dim=-1) >= grad_threshold
  selected_pts_mask &= self.get_scaling.max(dim=1).values <= percent_dense * scene_extent
  ```
  复制时直接复制原参数（`:572-578`），把新点放在同一位置。
- **split（分裂）**：点大且梯度大——这颗大高斯糊住了细节，要拆成 $N=2$ 颗小的。
  ```python
  selected_pts_mask = padded_grad >= grad_threshold
  selected_pts_mask &= self.get_scaling.max(dim=1).values > percent_dense * scene_extent
  ```
  分裂时新点在原高斯内部按其大小做高斯采样扰动（`:549-553`），新缩放变成原来的 $1/(0.8N)$（$N=2$，即缩小一圈）；不透明度也沿用原值（`:558`）。

> 注意两个分支都把 `new_objects_dc` 一起复制了（`:559`、`:578`）——身份编码必须跟着几何一起增减，否则新点没有身份。这是本项目相对原版的修改点之一。

### 6.4 prune（剪枝）

`densify_and_prune`（`:582`）在 clone/split 之后清理：
- 不透明度过低（`< min_opacity`）的点；
- 屏幕上半径过大的点（`max_radii2D > max_screen_size`）；
- 世界空间缩放过大的点（`> 0.1 * extent`）。

### 6.5 关键难点：同步修改优化器状态

密度化要往模型里加新点（clone/split）或删点（prune），这会改变每个 `_xxx` 张量的**长度**（第 0 维 $N$）。但 Adam 优化器里为每个参数维护了**动量状态** `exp_avg`、`exp_avg_sq`，它们的形状必须和参数本身一致。新手最困惑的就是：**张量改了长度，优化器状态怎么办？**

代码用几个函数解决，思路统一："改张量的同时，按相同方式改优化器状态"。

- `cat_tensors_to_optimizer`（`:496`）：新增点时，给每个参数组沿第 0 维 `torch.cat` 一段新值；同时给 `exp_avg`、`exp_avg_sq` 也 `cat` 一段**全零**的新状态（新点没有历史动量，从零开始）。
- `_prune_optimizer`（`:461`）：删点时，用一个布尔 `mask` 同时筛掉参数和它的动量状态（`:466-467`）。
- `replace_tensor_to_optimizer`（`:446`）：整体替换某参数（如 `reset_opacity` 时），把动量清零重置。
- `densification_postfix`（`:518`）：clone/split 的收尾，把新张量字典喂给 `cat_tensors_to_optimizer`，再把更新后的张量赋回 `self._xxx`。注意它也处理了 `obj_dc`（`:525`、`:534`）。

> 一句话：**凡是改了高斯数量，就必须同步改优化器状态，否则形状不匹配、训练直接崩。**

## 7. PLY 存取：save_ply / load_ply

训练好的高斯要存盘。3DGS 用 PLY 格式（一种简单的点云格式）。本项目对 PLY 字段做了一处扩展。

### 7.1 construct_list_of_attributes

`scene/gaussian_model.py:357` 列出每个顶点要存哪些字段，按顺序是：

```
x y z nx ny nz
f_dc_0 f_dc_1 f_dc_2
f_rest_0 ... f_rest_44
opacity
scale_0 scale_1 scale_2
rot_0 rot_1 rot_2 rot_3
obj_dc_0 ... obj_dc_15      ← 本项目新增
```

末尾多出的 `obj_dc_0..15` 共 16 个浮点，就是身份编码。每个 `_objects_dc` 形状是 $(N,1,16)$，存盘时 `transpose(1,2).flatten(start_dim=1)` 摊平成 $(N,16)$（`:383`）。

### 7.2 save_ply / load_ply

`save_ply`（`:373`）把上述字段拼成结构化 numpy 数组写盘。`load_ply`（`:398`）反向读回，按 `num_objects` 循环读取 `obj_dc_0..15` 还原成 $(N,16,1)$ 再转置成 $(N,1,16)$（`:432-434`、`:442`）。这样训练好的模型（含身份）就能完整保存与复现。

## 8. 渲染 render()

`gaussian_renderer/__init__.py:18` 的 `render()` 把一群高斯"泼"成图像。逐步拆解：

### 8.1 构造光栅化配置 raster_settings

```python
raster_settings = GaussianRasterizationSettings(
    image_height=..., image_width=...,
    tanfovx=tanfovx, tanfovy=tanfovy,       # 视锥半角的正切
    bg=bg_color, scale_modifier=...,
    viewmatrix=..., projmatrix=...,          # 相机矩阵
    sh_degree=pc.active_sh_degree,
    campos=viewpoint_camera.camera_center,   # 相机位置，给 SH 算视角用
    prefiltered=False, debug=pipe.debug)
```

这里把视锥（FoV）、世界到屏幕的变换矩阵、背景色、当前 SH 阶数等一股脑打包给光栅化器。`tanfovx = math.tan(FoVx * 0.5)` 来自相机的水平视场角（`:33`）。

### 8.2 准备各属性

```python
means3D = pc.get_xyz
opacity = pc.get_opacity                 # sigmoid 后
scales = pc.get_scaling                  # exp 后
rotations = pc.get_rotation              # normalize 后
shs = pc.get_features                    # (N,16,3)，DC+高阶在 dim=1 拼接
sh_objs = pc.get_objects                 # (N,1,16) 身份编码
```

注意 `means2D` 是一个全零、但 `requires_grad=True` 的张量（`:26`），目的是让 PyTorch 把"屏幕空间位置"的梯度传回来，供密度化统计用。

### 8.3 调用光栅化器

```python
rendered_image, radii, rendered_objects = rasterizer(
    means3D=means3D, means2D=means2D,
    shs=shs, sh_objs=sh_objs,           # ← 身份特征和颜色一起喂进去
    colors_precomp=colors_precomp, opacities=opacity,
    scales=scales, rotations=rotations, cov3D_precomp=cov3D_precomp)
```

（`gaussian_renderer/__init__.py:86`）返回三样东西：
- `rendered_image`：$(3,H,W)$ 的 RGB 图。
- `radii`：每个高斯在屏幕上的半径，用于判断是否可见（`radii > 0`）。
- **`rendered_objects`**：$(16,H,W)$ 的**身份特征图**！这正是本项目的核心输出之一。

身份特征图是怎么来的？和 RGB 用的是**同一套**前向投影 + alpha 合成：每颗高斯有一个 16 维的"身份向量"，按它在不透明度上的权重 $\alpha_i$ 从前到后叠加：

```math
\mathbf{O}(u,v) = \sum_{i\in\text{sorted}} \mathbf{o}_i\,\alpha_i\prod_{j<i}(1-\alpha_j)
```

其中 $\mathbf{o}_i$ 就是第 $i$ 颗高斯的 16 维身份向量。和颜色的合成公式完全同构，只是把 3 维换成了 16 维。因此它**天然可微**，能和颜色一起被监督学习。

### 8.4 三个开关

- `override_color`：若不为 `None`，直接用给定颜色，跳过 SH（常用于在白色背景下渲染纯色高斯）。
- `convert_SHs_python`：为 `True` 时用 Python/PyTorch 算 SH→RGB（`:73-78`），否则交给 CUDA 内核算，更快。
- `compute_cov3D_python`：为 `True` 时在 Python 端预算协方差（`:62-63`），否则在 CUDA 端算。

调试时打开 Python 版便于排查，正常训练都关掉走 CUDA。

## 9. 修改版光栅化器：本项目的灵魂改动

打开 `submodules/diff-gaussian-rasterization/diff_gaussian_rasterization/__init__.py`，对比官方 `diff-gaussian-rasterization`，你会发现本项目对它动了"手术"。

### 9.1 forward 多收一个、多返回一个

`_RasterizeGaussians.forward`（`:45`）的参数列表多了 `sh_objs`；它被塞进传给 C++ 内核的 `args`（`:76`），而 C++ 内核的返回值从原来的 `(num_rendered, color, radii, ...)` 变成了：

```python
num_rendered, color, objects, radii, geomBuffer, binningBuffer, imgBuffer = _C.rasterize_gaussians(*args)  # :87
```

多出来的 `objects` 就是 16 通道身份特征图。`forward` 返回 `(color, radii, objects)`（`:99`）。

### 9.2 backward 多了两路梯度

`backward`（`:102`）多接收一个 `grad_out_objects`（身份特征图回传的梯度），多返回一个 `grad_objects`（传给 `sh_objs` 的梯度，`:149` 还做了 `unsqueeze(1)` 对齐形状）。这意味着**身份特征能通过 alpha 合成管线把损失传回每颗高斯的 `_objects_dc`**，于是身份可以被监督学习。

### 9.3 含义与提醒

身份特征和颜色走完全相同的前向投影 + alpha 合成，因此可微、可端到端训练。这就是 Gaussian-Grouping 能让每颗高斯"知道自己属于哪个物体"的技术基础。

> **重要提醒**：因为光栅化器被改过，你**必须编译本仓库 `submodules/diff-gaussian-rasterization` 自带的版本**，不能用 `pip install diff-gaussian-rasterization` 装官方版，否则参数对不上、身份特征图根本出不来。同样的，`simple_knn`（在 `create_from_pcd` 里被 `from simple_knn._C import distCUDA2` 调用，`scene/gaussian_model.py:17`）也要编译仓库自带的 submodule。

## 10. 辅助数学函数速览

最后把散落在 `utils/` 里的几个关键数学工具列一下，方便对照。

### 10.1 球谐（`utils/sh_utils.py`）

- `eval_sh(deg, sh, dirs)`（`:57`）：给定 SH 系数 `sh`（形状 `[..., C, (deg+1)^2]`）和单位方向 `dirs`（`[...,3]`），按硬编码的多项式算出该方向的值（`[...,C]`）。阶数 0 就只有 $C_0 \cdot \text{sh}[...,0]$，阶数 1 加上 $C_1$ 与 $x,y,z$ 的项，依此类推到阶 4。$C_0=0.28209479$ 等。
- `RGB2SH(rgb) = (rgb - 0.5)/C0`（`:114`）：RGB 到 SH DC 空间的线性映射。
- `SH2RGB(sh) = sh*C0 + 0.5`（`:117`）：逆映射，把 DC 系数变回 RGB。

### 10.2 旋转与协方差（`utils/general_utils.py`）

- `inverse_sigmoid(x) = log(x/(1-x))`（`:18`）：sigmoid 的逆，用于把不透明度存成 logit。
- `get_expon_lr_func`（`:29`）：位置的指数衰减学习率调度器（对数空间线性插值，即实际空间指数衰减）。
- `build_rotation(r)`（`:78`）：四元数 $(w,x,y,z)$ → $3\times3$ 旋转矩阵 $R$，先把四元数归一化再用标准公式填 $R$。
- `build_scaling_rotation(s, r)`（`:101`）：构造 $L = R\cdot\text{diag}(s)$，用于协方差 $\Sigma = LL^\top$。

### 10.3 相机几何（`utils/graphics_utils.py`）

- `getWorld2View2(R, t, translate, scale)`（`:38`）：由相机旋转 `R` 和平移 `t` 构造世界→相机变换矩阵，并支持整体平移和缩放（归一化场景用）。
- `getProjectionMatrix(znear, zfar, fovX, fovY)`（`:51`）：透视投影矩阵，由近远裁剪面和视场角算出。
- `fov2focal` / `focal2fov`（`:73`/`:76`）：视场角与焦距互化，公式 `focal = pixels / (2*tan(fov/2))`。

## 11. 与原版 3DGS 的差异小结

| 维度 | 原版 3DGS | Gaussian-Grouping（本项目） |
|---|---|---|
| 高斯参数 | 6 组（xyz, f_dc, f_rest, scaling, rotation, opacity） | 多 1 组 `_objects_dc` 身份编码 $(N,1,16)$ |
| 初始化 | SfM 点云颜色 RGB2SH | 额外对随机身份做 `RGB2SH` 初始化 |
| 优化器参数组 | 6 组 | 7 组（`obj_dc` 复用 `feature_lr`） |
| 密度化 | clone/split/prune 6 组参数 | 同步处理 `obj_dc`（clone/split/postfix 都复制身份） |
| PLY 字段 | 标准 3DGS 字段 | 末尾多 `obj_dc_0..15` |
| 光栅化器 | 官方 `diff-gaussian-rasterization` | **改版**：forward 收 `sh_objs` 返 `objects`，backward 收 `grad_out_objects` 返 `grad_objects` |
| 渲染输出 | RGB + radii | RGB + radii + **16 通道身份特征图** |
| 编译要求 | 官方 submodule 即可 | 必须编译仓库自带改版 submodule（含 `simple_knn`） |

## 小结

这一节我们把"模型"和"渲染"两块代码彻底拆开看了一遍。`GaussianModel` 是一个装着几百万颗高斯的普通容器，每颗高斯带 7 组参数（含本项目新增的身份编码）；这些参数被设计成"激活前的值"以便无约束优化；密度化在加减点时会同步维护优化器状态；改版光栅化器让 16 维身份和 3 维颜色走同一条可微 alpha 合成管线，最终同时输出 RGB 图和身份特征图。理解了这些，下一节我们就可以进入训练循环，看损失函数如何把"身份特征图"和"伪标签（pseudo label）"对齐，让每颗高斯真正学会"我属于哪个物体"。
