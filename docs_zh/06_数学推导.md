## 1. 一个 3D 高斯长什么样

回顾第 2 篇的比喻：**3D 高斯像一颗半透明、可变形、会发光的水母**。它的"密度"在中心最高，向四周按钟形衰减。数学上，给定空间一点 $x\in\mathbb R^3$，高斯在 $x$ 处的"未归一化密度"为：

```math
G(x)=\exp\!\left(-\tfrac12\,(x-\mu)^\top \Sigma^{-1}(x-\mu)\right)
```

- $\mu$ 是水母的"心脏"——中心位置，决定它漂在哪儿。
- $\Sigma$ 是 $3\times3$ 的协方差矩阵，决定水母的**形状**（胖瘦）与**朝向**（怎么扭）。可以证明 $\Sigma$ 的特征向量给出椭球的三根主轴方向，特征值的平方根给出三根半轴长度。

> **一个常被忽略的细节**：标准的多元高斯概率密度前面有个归一化系数 $\dfrac{1}{(2\pi)^{3/2}|\Sigma|^{1/2}}$。3DGS **故意不存这个系数**。原因有二：(1) 渲染时我们只关心不同点的相对贡献，最终又会被不透明度 $\alpha$ 和 over 操作重新归一化；(2) 省掉它就少一个需要维护的量。所以你看到的 $G(x)$ 是"裸"指数部分。

那么 $\Sigma$ 怎么存？这就是下一节的核心。

## 2. 协方差矩阵的参数化（关键推导）

### 2.1 直接优化 $\Sigma$ 有什么问题

3DGS 要用梯度下降去优化 $\Sigma$，让它逐步变成"正确的形状"。但直接把 $\Sigma$ 当 9 个自由参数来优化，有两个硬伤：

1. **必须保证半正定（positive semi-definite, PSD）**。半正定等价于"对所有非零向量 $v$，都有 $v^\top\Sigma v\ge 0$"。几何上，这保证 $G(x)\le 1$（指数里非正），否则高斯会"爆炸"成无穷大。梯度下降一次更新后，$\Sigma$ 很可能就不再半正定了。
2. **6 个独立参数有耦合**。对称的 $\Sigma$ 有 6 个独立元素，但它们和"形状/朝向"的关系是纠缠的，优化起来又慢又难。

### 2.2 3DGS 的妙招：$\Sigma = RSS^\top R^\top$

3DGS 不直接存 $\Sigma$，而是存**缩放 $s$** 和**四元数 $q$**，再合成 $\Sigma$：

```math
\Sigma = R\,S\,S^\top R^\top,\qquad S=\mathrm{diag}(s_1,s_2,s_3)
```

其中 $R$ 是由四元数 $q$ 构造的旋转矩阵。把 $S$ 拆开看：$S$ 是对角矩阵，$s_i>0$ 给出三根半轴长度；$R$ 把这根"沿坐标轴的椭球"转到空间中的任意朝向。

**几何直觉**：先在原点放一个三轴分别长 $s_1,s_2,s_3$ 的椭球（$S$），再用 $R$ 把它转到想要的方向，最后平移到 $\mu$。

### 2.3 半正定自动满足（证明）

令 $M=RS$，则 $\Sigma=MM^\top$。对任意非零 $v\in\mathbb R^3$：

```math
v^\top\Sigma\,v = v^\top MM^\top v = (M^\top v)^\top(M^\top v)=\|M^\top v\|^2 \ge 0
```

最后一步用了"向量长度的平方非负"。**只要 $s_i>0$，半正定就自动成立**，再也不用在每步优化后做投影。

> **那为什么还要保证 $s_i>0$？** 代码里存的是 $\log s$，激活时用 $s=\exp(\log s)$，指数函数恒正，所以这条约束也自动满足。这就是 `scaling_activation = torch.exp`（`scene/gaussian_model.py:31`）的作用。

### 2.4 四元数 → 旋转矩阵

四元数 $q=(w,x,y,z)$（代码里写 $r=(q_0,q_1,q_2,q_3)$，先归一化为单位四元数）。对应的旋转矩阵 $R$（对应 `utils/general_utils.py:78-99` 的 `build_rotation`）：

```math
R=\begin{pmatrix}
1-2(y^2+z^2) & 2(xy-wz) & 2(xz+wy)\\
2(xy+wz) & 1-2(x^2+z^2) & 2(yz-wx)\\
2(xz-wy) & 2(yz+wx) & 1-2(x^2+y^2)
\end{pmatrix}
```

> 代码里变量名是 `r=q[:,0]=w`、`x=q[:,1]`、`y=q[:,2]`、`z=q[:,3]`，把上式逐行写进去（`utils/general_utils.py:90-98`）。开头先 `q = r / norm` 归一化（`:81`），保证它是单位四元数。优化过程中四元数可能漂移出单位长度，所以每次取用时通过 `rotation_activation = normalize`（`scene/gaussian_model.py:39`）再拉回单位球。

### 2.5 与代码逐行对应

`build_scaling_rotation`（`utils/general_utils.py:101-110`）做了两步：

1. $L=\mathrm{diag}(s)$（`:102-107`）；
2. $L \leftarrow R\,L$（`:109`），即 $L=RS=M$。

`build_covariance_from_scaling_rotation`（`scene/gaussian_model.py:25-29`）再算：

```math
\Sigma = L\,L^\top = (RS)(RS)^\top = RSS^\top R^\top
```

这就是 `actual_covariance = L @ L.transpose(1,2)`（`:27`）。注意调用时还乘了一个 `scaling_modifier`（默认 1），用于渲染时统一缩放所有高斯。`strip_symmetric` 把 $3\times3$ 压成 6 个独立分量传给 CUDA。

## 3. 视角相关颜色：球谐函数（SH）

### 3.1 为什么需要 SH

真实物体有反光：从不同角度看，颜色会变（金属、玻璃、湿润表面尤其明显）。如果每个高斯只存一个固定 RGB，模型表达不出这种"随视角变化"。

**球谐函数**是一组定义在单位球面上的正交基底函数 $Y_l^m(d)$（$l$ 是阶，$m$ 是该阶内的序号）。任何定义在球面上的函数都可以展成它们的线性组合——就像傅里叶级数把周期函数展成正弦余弦。3DGS 把"颜色随观察方向 $d$ 变化"的函数这样展开：

```math
c(d)=\sum_{l=0}^{L}\sum_{m=-l}^{l} a_{l,m}\,Y_l^m(d)
```

- $d$ 是观察方向（单位向量）。
- $a_{l,m}$ 是要学的系数。阶数 $L$ 越高，细节越丰富，但参数也越多。

3DGS 默认用 $L\le 3$（`sh_degree = 3`，`arguments/__init__.py:46`）。每阶有 $2l+1$ 个 $m$，所以总系数数 = $\sum_{l=0}^{3}(2l+1)=(l_{\max}+1)^2=4^2=16$ 个。**每通道**（R/G/B）16 个，所以一颗高斯共 $16\times3=48$ 个 SH 系数。

### 3.2 代码里怎么存

`create_from_pcd`（`scene/gaussian_model.py:134`）分配 `features` 为 $(N, 3, 16)$，其中前 3 列通道存第 0 阶系数、其余初始化为 0（`:135-136`）。随后做转置拆开存放：

- `_features_dc` 存第 0 阶（常数项 $Y_0^0$）的系数，即 `features[:,:,0:1].transpose(1,2)`，存储形状为 $(N,1,3)$（`:152`）；
- `_features_rest` 存其余 15 个高阶系数 `features[:,:,1:].transpose(1,2)`，形状 $(N,15,3)$（`:153`）。

> 拆开后第一维是"系数数"，第二维才是 R/G/B 通道——这是为了和光栅化器的输入约定对齐。把常数项单独拆出来，是因为它代表"平均颜色"，初始化和优化都更稳定（`_features_rest` 的学习率还要再除以 20，`scene/gaussian_model.py:168`）。

### 3.3 RGB ↔ SH 互转

零阶球谐 $Y_0^0(d)=C_0=\dfrac{1}{2}\sqrt{\dfrac{1}{\pi}}\approx0.28209479177387814$（`utils/sh_utils.py:26`）。它是个**常数**，不依赖方向。

初始化时已知点云颜色 RGB，要反推零阶系数：

```math
a_{0,0}=\mathrm{RGB2SH}(\text{rgb})=\frac{\text{rgb}-0.5}{C_0}
```

反过来：

```math
\text{rgb}=\mathrm{SH2RGB}(a_{0,0})=a_{0,0}\cdot C_0+0.5
```

（`utils/sh_utils.py:114-118`）

**为什么有 $-0.5/+0.5$？** 因为 RGB 取值范围是 $[0,1]$，而 SH 系数希望以 0 为中心、可正可负（方便梯度优化）。$0.5$ 是把 $[0,1]$ 平移到 $[-0.5,+0.5]$ 的中点。除/乘 $C_0$ 是为了把零阶项的真实数值（约 $C_0$）规整到 $a_{0,0}\approx1$ 的量级。

### 3.4 eval_sh 的零阶与一阶展开

`eval_sh(deg, sh, dirs)`（`utils/sh_utils.py:57-112`）按方向 $d=(x,y,z)$ 计算颜色。零阶和一阶展开为：

```math
c(d)=C_0\,a_{0,0}\;-\;C_1\,y\,a_{1,-1}\;+\;C_1\,z\,a_{1,0}\;-\;C_1\,x\,a_{1,1}\;+\;\cdots
```

对应代码 `:74-80`。这里的 $d$ 是什么？看 `gaussian_renderer/__init__.py:75-76`：

```python
dir_pp = (pc.get_xyz - camera_center)       # 从相机指向高斯中心的向量
dir_pp_normalized = dir_pp / dir_pp.norm(dim=1, keepdim=True)  # 归一化为单位向量 d
```

所以 $d$ 是"**从相机指向高斯中心**"的单位向量——这正是观察方向。

> 小提示：3DGS 里 SH 也可以在 Python 里算（`convert_SHs_python=True`），但默认交给 CUDA 在光栅化时一起算，更快。两种方式数学等价。注意 Python 路径里会再 `+0.5` 并 clamp 到非负（`__init__.py:78`），因为 `eval_sh` 输出的是"减 0.5 后的 SH 颜色"。

## 4. 从 3D 到 2D：EWA splatting

### 4.1 思想

渲染要把 3D 高斯画到 2D 屏幕上。直接"投影每个点"会失真，因为高斯是个连续分布。EWA（Elliptical Weighted Averaging）splatting 的核心结论是：**3D 高斯经过（局部仿射近似的）视图+投影变换后，在屏幕上仍然是 2D 高斯**。我们只需算出这个 2D 高斯的中心和协方差。

### 4.2 仿射变换保持高斯性

先证明一个引理：若随机变量 $X$ 服从均值 $\mu$、协方差 $\Sigma$ 的高斯，对它做仿射变换 $y=Ax+b$，则 $Y=AX+b$ 仍服从高斯，均值为 $A\mu+b$，协方差为：

```math
\mathrm{Cov}(Y)=A\,\Sigma\,A^\top
```

**证明**：$\mathrm{Cov}(Y)=\mathbb E[(Y-\mathbb E Y)(Y-\mathbb E Y)^\top]=\mathbb E[(A(X-\mu))(A(X-\mu))^\top]=A\,\mathbb E[(X-\mu)(X-\mu)^\top]\,A^\top=A\Sigma A^\top$。

### 4.3 视图+投影的局部仿射近似

把视图变换记为 $W$（世界→相机），投影雅���比记为 $J$（在 $\mu$ 处对针孔投影做一阶泰勒展开）。组合起来，$\mu$ 附近的映射近似为：

```math
\mathcal T(x)\approx J\,W\,x + t
```

这是一个仿射映射 $y=Ax+b$，其中 $A=JW$。代入引理，2D 协方差：

```math
\Sigma' = J\,W\,\Sigma\,W^\top J^\top
```

2D 高斯在像素 $u$ 处的值：

```math
G'(u)=\exp\!\left(-\tfrac12\,(u-\mu')^\top{\Sigma'}^{-1}(u-\mu')\right)
```

- $\mu'$ 是 $\mu$ 投影到屏幕的 2D 中心；
- $\Sigma'$ 是 $2\times2$ 的 2D 协方差，描述屏幕上"水母投影"的椭圆形状。

**对应代码**：`gaussian_renderer/__init__.py:43-44` 把 `viewmatrix` 与 `projmatrix`（=视图×投影）一起传给光栅化器；`tanfovx`/`tanfovy`（`:33-34`）参与针孔模型的雅可比计算。$J$ 的解析表达式在 CUDA 内部，按标准针孔相机模型 $\Pi(X,Y,Z)=(f_x X/Z,\,f_y Y/Z)$ 求导得到。

> 这一步"仿射近似"是 3DGS 快的根源：避免了逐像素沿光线积分（NeRF 的做法），而是把每颗高斯"啪"一下拍成屏幕上一个椭圆，再做合成。

## 5. Alpha 合成（可微体渲染近似）

### 5.1 单个高斯在像素 $u$ 的不透明度

第 $i$ 颗高斯在像素 $u$ 处贡献的"不透明度"是两个因子相乘：

```math
\alpha_i(u)=o_i\cdot G'_i(u)
```

- $o_i=\sigma(\texttt{\_opacity}_i)\in(0,1)$：高斯自身的"基础不透明度"，代码里 `get_opacity = sigmoid(_opacity)`（`scene/gaussian_model.py:120-121`）。
- $G'_i(u)$：上一节算出的 2D 高斯在 $u$ 处的值（已经隐含了由 $\Sigma'$ 决定的椭圆衰减）。

### 5.2 从前到后的 over 操作

把所有覆盖到 $u$ 的高斯按深度**从前到后排序**，记为 $i=1,2,\dots,N$。像素颜色按"over"操作合成（像一层层半透明颜料叠上去）：

```math
C(u)=\sum_{i=1}^{N} c_i\,\alpha_i\,\prod_{j=1}^{i-1}(1-\alpha_j)
```

- $c_i$：第 $i$ 颗高斯的颜色（视角相关，由 SH 给出）；
- $\alpha_i$：上面算出的不透明度；
- $\prod_{j<i}(1-\alpha_j)$：**透射率**，即"前面所有高斯没挡住、让光透过来的比例"。越靠后，这个连乘越小，该高斯能贡献的颜色越少——这就是遮挡。

**对应代码**：`rendered_image` 就是 $C(u)$ 逐像素计算的结果（`gaussian_renderer/__init__.py:86`）。

### 5.3 为什么"可微"

整个 $C(u)$ 对 $\mu_i,\Sigma_i,o_i,c_i$ 都是可微的（指数、乘法、连乘都光滑）。所以一旦拿到渲染图与真实图的误差，**误差的梯度能顺着公式一路反传到每颗高斯的每个参数**——这就是"可微体渲染"。PyTorch 的 `loss.backward()`（`train.py:106`）正是基于此更新所有 `_xxx` 参数。

## 6. 身份编码的可微渲染（本项目核心）

到这里为止，都是原版 3DGS。Gaussian-Grouping 的关键创新是：**给每颗高斯额外挂一个 16 维"身份编码"**，并用同一套 α 合成把它渲染成一张"身份特征图"。

### 6.1 给每颗高斯挂身份

对第 $i$ 颗高斯，额外存：

```math
f_i^{obj}\in\mathbb R^{16}
```

代码里就是 `_objects_dc`（`scene/gaussian_model.py:51`），维度由 `num_objects = 16`（`:52`）规定。初始化时用 `RGB2SH(torch.rand(...))` 随机生成（`:139`），形状再补成 $(N,1,16)$，随后被优化。注意初始化经过 `RGB2SH` 只是借用那个 $(r-0.5)/C_0$ 变换，把随机数映射到以 0 为中心的范围，并不是真的要当球谐用。

> 注意它的命名 `_objects_dc` 借用了 SH 的 "dc"（direct current，常数项）写法，存储形状也是 $(N,1,16)$，但**它不是球谐**——它只是一个静态的 16 维向量，不随视角变化。这样命名并保持第一维为 1，是为了让修改版光栅化器能"假装"它是 SH 一样复用同一套渲染管线。

### 6.2 复用 α 合成，把颜色换成身份

关键洞察：α 合成公式 $C(u)=\sum_i c_i\alpha_i\prod_{j<i}(1-\alpha_j)$ 里，**颜色 $c_i$ 是个 3 维向量**。如果把它换成 16 维的身份向量 $f_i^{obj}$，公式形式完全不变——只是每像素输出从 3 通道变成 16 通道：

```math
F^{obj}(u)=\sum_{i=1}^{N} f_i^{obj}\,\alpha_i\,\prod_{j=1}^{i-1}(1-\alpha_j)\;\in\mathbb R^{16}
```

得到的 $F^{obj}$ 是一张 $16\times H\times W$ 的"**身份特征图**" `rendered_objects`——光栅化器的第三个返回值（`gaussian_renderer/__init__.py:86`），对应传入的 `sh_objs = pc.get_objects`（`:81`、`:90`）。

### 6.3 为什么这是优雅的设计

因为**复用了同一套 $\alpha_i$**（同一套排序、同一个 $o_i$、同一个 $\Sigma'$），所以原版 CUDA 光栅化器只需改一处：把"取 3 维颜色做加权"改成"取 16 维身份做加权"，其余逻辑（投影、排序、遮挡）一行不动。**这就是修改版 `diff-gaussian-rasterization` 干的事**——支持 `sh_objs` 输入，输出第三张图。

> 物理含义：$F^{obj}(u)$ 是"看到像素 $u$ 的那束光里，混入了哪些物体的身份信息，按不透明度加权平均"。同一物体表面的像素，身份向量会很接近；不同物体的边界处，身份向量会突变。

## 7. 分类器与 2D 分组损失

### 7.1 1×1 卷积分类器

身份特征图是 16 维的，但我们要的是"这个像素属于哪个物体"——一个类别。所以接一个**逐像素**的小分类器 $g_\theta$：

```python
classifier = torch.nn.Conv2d(gaussians.num_objects, num_classes, kernel_size=1)   # train.py:33
```

$1\times1$ 卷积等价于"对每个像素独立地做一个 $16\to C$ 的线性层"。输入 $F^{obj}$（$16\times H\times W$），输出 logit $z\in\mathbb R^{C\times H\times W}$，再 softmax 得概率：

```math
p(\cdot|u)=\mathrm{softmax}\!\big(g_\theta(F^{obj}(u))\big)\in\mathbb R^{C},\qquad \sum_{c=1}^{C}p_c=1
```

其中类别数 $C$ 由 `num_classes` 控制，默认 200（`arguments/__init__.py:58`）。分类器自身用 Adam 单独优化，学习率 $5\times10^{-4}$（`train.py:35`）。

### 7.2 与 SAM/DEVA 伪标签做交叉熵

每个训练视角带一张由 SAM/DEVA 生成的 2D 分割伪标签（pseudo label）$y(u)\in\{1,\dots,C\}$。逐像素交叉熵：

```math
\mathcal L_{obj}=\frac{1}{HW}\sum_{u}\mathrm{CE}\!\big(p(\cdot|u),\,y(u)\big)=-\frac{1}{HW}\sum_u\log p_{y(u)}(u)
```

代码（`train.py:88-90`）：

```python
logits = classifier(objects)                                  # (C, H, W)
loss_obj = cls_criterion(logits.unsqueeze(0), gt_obj.unsqueeze(0)).squeeze().mean()
loss_obj = loss_obj / torch.log(torch.tensor(num_classes))    # 归一化到 (0,1)
```

> **为什么要除以 $\log C$？** 均匀猜测的概率是 $1/C$，对应的交叉熵恰是 $\log C$。除以它就把损失"校准"到 $(0,1)$ 区间，方便和其他损失项加权时调参（`train.py:90`）。

梯度路径：$\mathcal L_{obj}$ → 分类器 → $F^{obj}$（16 通道身份图）→ 每颗高斯的 $f_i^{obj}$。也就是说，**2D 的伪标签能监督 3D 中每颗高斯学会"自己是谁"**。

## 8. 3D 空间一致性正则

### 8.1 动机

只靠 2D 伪标签监督有个隐患：那些**在训练视角里被遮挡的 3D 区域**，从未被任何像素采样到，它们的 $f_i^{obj}$ 学不到东西，可能学成"四不像"。我们需要一个**纯 3D 的先验**：空间上相邻的点，应当属于同一物体，身份应当一致。

### 8.2 概率化身份

先把每颗高斯的身份向量也送进同一个分类器，得到它在 $C$ 类上的概率分布：

```python
logits3d = classifier(gaussians._objects_dc.permute(2,0,1))          # train.py:99
prob_obj3d = torch.softmax(logits3d, dim=0).squeeze().permute(1,0)    # (N, C)
```

得到 $p_i\in\mathbb R^C$（$\sum_c p_{i,c}=1$），表示第 $i$ 颗高斯属于各类的概率。

### 8.3 k 近邻 + KL 散度

对每个 3D 点 $i$，在 **xyz 空间**找它的 $k$ 近邻 $\mathcal N_k(i)$（默认 $k=5$，`reg3d_k`）。约束"$i$ 和邻居的身份分布应一致"，用 KL 散度：

```math
\mathrm{KL}(p_i\,\|\,p_j)=\sum_{c=1}^{C}p_{i,c}\,\log\frac{p_{i,c}}{p_{j,c}}
```

KL 越小，两个分布越接近；为 0 则完全相同。对所有采样点及其邻居平均：

```math
\mathcal L_{3d}=\frac{\lambda}{C}\cdot\frac{1}{|S|}\sum_{i\in S}\frac{1}{k}\sum_{j\in\mathcal N_k(i)}\mathrm{KL}(p_i\,\|\,p_j)
```

- $S$：随机采样的点集（`reg3d_sample_size`，默认 1000）；当总点数超过 `reg3d_max_points`（默认 300000）时先随机降采样再采样；
- $\lambda$：`reg3d_lambda_val`（默认 2）；
- 除以 $C$ 把量级归一化。

> 注意 `loss_cls_3d` 的函数签名里 `max_points` 默认是 200000、`sample_size` 默认 800，但 `train.py` 调用时显式传入了 `reg3d_max_points=300000` 与 `reg3d_sample_size=1000`，实际跑的是后者。

### 8.4 与 loss_cls_3d 代码逐行对应

`utils/loss_utils.py:74-115`：

| 公式部分 | 代码 |
|---|---|
| 先按 `max_points` 降采样 | `if features.size(0) > max_points: indices = randperm`（`:89-92`） |
| 随机采样 `sample_size` 个点 | `indices = torch.randperm(...)[:sample_size]`（`:96-98`） |
| 求 $k$ 近邻 | `dists = torch.cdist(sample_features, features)`（`:101`），`dists.topk(k, largest=False)`（`:102`） |
| 取邻居预测 | `neighbor_preds = predictions[neighbor_indices_tensor]`（`:105`） |
| $\mathrm{KL}(p_i\|p_j)$ | `sample_preds.unsqueeze(1) * (log(sample_preds+1e-10) - log(neighbor_preds+1e-10))`（`:108`）—— 正是 $p_i(\log p_i-\log p_j)$ |
| 求和+平均 | `kl.sum(dim=-1).mean()`（`:109`） |
| 除以 $C$ | `normalized_loss = loss / num_classes`（`:113`） |
| 乘 $\lambda$ | `return lambda_val * normalized_loss`（`:115`） |

> **xyz 被 detach**：调用时传入 `gaussians._xyz.squeeze().detach()`（`train.py:101`）。这意味着这个正则**只用来决定"谁是邻居"，并调整身份编码，不动高斯位置**——否则它会为了"凑一致"而把点挪走，破坏几何。

## 9. 重建损失

身份学得再好，画面崩了也没用。RGB 重建损失保证渲染图与真实照片一致：

```math
\mathcal L_{rgb}=(1-\lambda_{dssim})\,\mathcal L_1(\hat I,I)+\lambda_{dssim}\,\big(1-\mathrm{SSIM}(\hat I,I)\big)
```

- $\mathcal L_1(\hat I,I)=\dfrac{1}{HW\cdot3}\sum|\hat I-I|$（`l1_loss`，`utils/loss_utils.py:15-16`）：逐像素绝对差，保证**颜色准**，但容易糊。
- $\mathrm{SSIM}$（结构相似性，`ssim`，`:41-71`）：在 $11\times11$ 高斯窗口内比较亮度、对比度、结构，更贴近人眼对**纹理/结构**的感知。取 $1-\mathrm{SSIM}$ 作为损失。
- $\lambda_{dssim}=0.2$（`arguments/__init__.py:85`，运行时即 `opt.lambda_dssim`）：L1 占大头，SSIM 做补充。两者互补：L1 防止整体偏色，SSIM 防止高频细节被抹平。

对应 `train.py:102` / `:104`：

```python
loss = (1-opt.lambda_dssim)*Ll1 + opt.lambda_dssim*(1-ssim(image, gt_image)) + ...
```

## 10. 密度化的判据

密度化（densification）是 3DGS 的"哪里细节不够就在哪里再撒一把点"机制。它的判据全靠一个量：**累积的屏幕空间位置梯度** $\bar g_i$。

### 10.1 累积梯度

每次渲染后，记录每个高��� 2D 中心的梯度（取前两维 x、y 的范数），按可见次数平均（`scene/gaussian_model.py:598-600` 累加，`densify_and_prune` 里 `:583` 求平均）：

```math
\bar g_i=\frac{\texttt{xyz\_gradient\_accum}_i}{\texttt{denom}_i}
```

$\bar g_i$ 大，意味着"这个高斯在屏幕上一直被推开"——它在努力弥补重建缺口，说明**这里需要更多高斯**。

### 10.2 clone 还是 split

阈值 $\tau_{grad}$（`densify_grad_threshold`，默认 `0.0002`，`arguments/__init__.py:90`）。再按当前尺度分情况（`densify_and_clone` / `densify_and_split`，`:566`/`:540`）：

| 条件 | 操作 | 含义 |
|---|---|---|
| $\max s_i\le\rho\cdot E$ 且 $\|g_i\|\ge\tau_{grad}$ | **clone**：复��一份，位置与原点相同 | 小高斯在"欠采样"区，复制它填补空洞 |
| $\max s_i>\rho\cdot E$ 且 $\bar g_i\ge\tau_{grad}$ | **split**：一分为 $N=2$ 份，缩放除以 $0.8N$ | 大高斯在"过度扩张"，把它拆小，各管一块 |

- $\rho$ = `percent_dense`（默认 0.01，`arguments/__init__.py:84`），$E$ = `scene.cameras_extent`（场景尺度，相机平均距离）。
- clone 直接复制原参数（`densify_and_clone`，`:572-578`），新点与原点重合，靠后续梯度把它们拉开。
- split 时新缩放 $s' = s/(0.8N)$，即 `new_scaling = scaling_inverse_activation(get_scaling / (0.8*N))`（`:554`），在 log 空间等价于 $\log s'=\log s-\log(0.8N)$。新位置在原高斯内按其 $\Sigma$ 高斯采样：先按缩放 `stds` 采样、再用旋转 `rots` 旋回、最后加到原中心（`:549-553`），保证新点落在原椭球内。

### 10.3 剪枝

`densify_and_prune`（`:582`）删掉三类高斯：

1. $o_i<\tau_o$（`min_opacity=0.005`，调用处 `train.py:133`）：太透明，没用（`:589`）；
2. 屏幕半径 > `max_screen_size`（=20，`train.py:132`）：占地太大（`:591`）；
3. $\max s_i>0.1\,E$：3D 中过大（`:592`）。

每轮密度化后还会 `reset_opacity`（`:393-396`），把所有不透明度压到 0.01 以下——逼着网络重新决定哪些高斯该"亮起来"，避免某些点一直占着不干活。

## 11. 修复阶段的损失（简述）

编辑流程里"删除某物体后补洞"的修复（inpaint）阶段，会把场景拆成"旧点 + 新撒点"两部分：旧点 `requires_grad=False` 冻结，只在 mask 区域内新撒点并让它们可学（`scene/gaussian_model.py:322-328` 的拼接处）。损失只在 mask 内的新点上算：

```math
\mathcal L_{inpaint}=(1-\lambda)\,\mathcal L_1^{\text{mask内}}(\hat I,\,I_{inp})+\lambda\,\mathrm{LPIGS}(\hat I,\,I_{inp})
```

- $I_{inp}$ 是用图像修复模型（如 LaMa）补出来的伪 GT 图；
- L1 保证像素级接近；
- **LPIPS**（Learned Perceptual Image Patch Similarity）用预训练 VGG 的深层特征计算距离，关注"感知自然度"——让补出来的区域纹理真实，不只是数值对。这能避免补洞区出现塑料感或糊块。

## 12. 完整优化目标

把以上各项按发生条件拼起来，主训练循环的总损失为：

```math
\boxed{\;\mathcal L_{total}=\underbrace{(1-\lambda_{dssim})\mathcal L_1+\lambda_{dssim}(1-\mathrm{SSIM})}_{\mathcal L_{rgb}\,\text{ 重建}}\;+\;\underbrace{\mathcal L_{obj}}_{\text{2D 分组}}\;+\;\mathbb 1\,[t\bmod t_{int}=0]\,\underbrace{\mathcal L_{3d}}_{\text{3D 一致性}}\;}
```

对应 `train.py:97-104`：

```python
if iteration % opt.reg3d_interval == 0:
    loss = (1-λ_dssim)*Ll1 + λ_dssim*(1-ssim) + loss_obj + loss_obj_3d
else:
    loss = (1-λ_dssim)*Ll1 + λ_dssim*(1-ssim) + loss_obj
```

- $\mathcal L_{rgb}$：让渲染图逼近真实照片（几何+外观对）。
- $\mathcal L_{obj}$：让 2D 身份图能被 SAM/DEVA 伪标签正确分类（学会"我是谁"）。
- $\mathcal L_{3d}$（每 $t_{int}=2$ 步触发一次，`reg3d_interval=2`）：让 3D 相邻点身份一致，补全被遮挡区域。

每项各司其职：**重建管"长得像"，2D 分组管"分得对"，3D 正则管"藏起来的也要对"**。三者合力，最终得到一组既能在任意视角渲染出清晰照片、又能在 3D 中精确分组的高斯。

## 小结

- 3D 高斯的形状由 $\mu$（位置）和 $\Sigma=RSS^\top R^\top$（缩放×旋转×旋转ᵀ×缩放）决定；这种参数化**自动保证半正定**，是 3DGS 能稳定优化的基石。
- 球谐函数把"颜色随视角变化"展成 16 个系数/通道；RGB↔SH 通过常数 $C_0$ 和 $0.5$ 偏移互转。
- EWA splatting 用仿射近似把 3D 高斯拍成 2D 高斯，协方差变换 $\Sigma'=JW\Sigma W^\top J^\top$；再用从前到后的 α 合成得到可微的渲染图。
- Gaussian-Grouping 在每颗高斯上挂 16 维身份编码，**复用同一套 α 合成**渲染出 16 通道身份特征图，再用 1×1 卷积分到类别，与伪标签做交叉熵。
- 完整损失 = 重建 + 2D 分组 + （间歇）3D 一致性正则，三者共同把一组高斯训成可分组的新视角表示。
